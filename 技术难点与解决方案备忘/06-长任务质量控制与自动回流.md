## 业务背景

用户期望 Agent 能连续生成章节,但长篇小说不能只追求"写出来"。每章至少要满足大纲符合度、人物一致性、伏笔处理、节奏和风格要求,还要能经受后续 `/审核`。

## 技术难点

LLM 一次生成不稳定。若完全依赖用户审稿,用户成本太高;若无限自动重试,会造成费用失控和结果震荡。审核节点还要避免把轻微问题升级成整章重写。

## 解决方案(设计稿,待 LangGraph 落地)

采用"三道门 + 有上限回流":

1. WriterNode 内部 self-critique:生成后按 5 个维度评分。
2. 子代理审核:校对、历史顾问、综合审核分别检查专业问题。
3. 跨章节审计:用户在卷尾或重大节点通过 `/跨审` 手动触发。

每章生成最多进行 2 次内部修订,整章回流最多 3 次。允许 1 个低风险维度未达标,交给 review_gate 兜底,避免流程卡死。

## 最小 demo / 伪代码

```python
PASS_SCORE = 7
MAX_REVISE = 2


def pass_overall(scores: dict[str, int]) -> bool:
    passed_dimensions = sum(score >= PASS_SCORE for score in scores.values())
    return passed_dimensions >= 4


def write_with_self_critique(state: dict) -> dict:
    draft = call_writer(state)
    best_draft = draft

    for retry in range(MAX_REVISE):
        report = self_critique(best_draft, state)
        if pass_overall(report["scores"]):
            return {"draft": best_draft, "self_critique": report}

        best_draft = revise_draft(
            draft=best_draft,
            suggestions=report["suggestions"],
            state=state,
        )

    return {"draft": best_draft, "self_critique": report}
```

## 核心依赖版最小代码

```python
from pydantic import BaseModel, Field
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI


class DimScore(BaseModel):
    score: int = Field(ge=0, le=10)
    issues: list[str] = Field(default_factory=list)
    suggestion: str = ""


class SelfCritiqueReport(BaseModel):
    outline_fidelity: DimScore
    character_voice: DimScore
    foreshadow_handling: DimScore
    rhythm: DimScore
    style_match: DimScore


critique_prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是长篇小说章节质检器,请按五个维度输出结构化评分。"),
        ("human", "章节正文:\n{draft}\n\n上下文:\n{context}"),
    ]
)


def critique_with_langchain(draft: str, context: str) -> SelfCritiqueReport:
    llm = ChatOpenAI(model="qwen-plus", temperature=0)
    chain = critique_prompt | llm.with_structured_output(SelfCritiqueReport)
    return chain.invoke({"draft": draft, "context": context})
```

## 落地建议

- 定义 `SelfCritiqueReport`,包含每个维度分数、问题和一句修订建议。
- `pass_overall` 规则采用至少 4 个维度达到 7/10。
- `review_gate` 只对硬性问题触发回流,如字数严重不足、人物事实冲突、历史关键错误。
- 每次重试记录原因、模型档位、token 成本和最终采纳版本。

## 验收标准

- 低质量初稿会自动触发一次修订。
- 连续失败不会无限重试,会给用户可理解的介入提示。
- 审核报告能指出具体维度和修订建议。
- `/跨审` 可以基于章节摘要检查人物弧光、伏笔收支、信物流转和时间线。
