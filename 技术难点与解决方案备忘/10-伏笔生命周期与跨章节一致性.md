# 伏笔生命周期与跨章节一致性

## 业务背景

长篇小说需要持续管理伏笔。设计文档要求 `伏笔/伏笔库.md` 记录伏笔 ID、状态、埋设章节、回收章节、关联人物和物品,写作时自动注入相关伏笔,审核时检查是否按计划回收。

## 技术难点

伏笔是跨章节状态,不是单次文本生成问题。它可能经历“创建、潜伏、触发、已回收、弃用”等状态。若只靠 LLM 临场记忆,容易出现忘记回收、重复回收、提前揭示或与人物/信物流转冲突。

## 解决方案

将伏笔库作为结构化正典管理,Markdown 只是展示格式。内部维护 `ForeshadowItem`:

- `id`:自增唯一 ID。
- `title`、`description`。
- `status`:潜伏、触发、已回收、弃用。
- `plant_chapter`、`reveal_chapter`。
- `related_personas`、`related_items`。
- `notes`。

写作前由 `prep_context` 根据章节 ID 查询待埋设和待回收伏笔。写作后由审核节点检查正文是否实际处理。跨章节审计汇总范围内伏笔收支。

## 最小 demo / 伪代码

```python
from dataclasses import dataclass
from typing import Literal


ForeshadowStatus = Literal["潜伏", "触发", "已回收", "弃用"]


@dataclass
class ForeshadowItem:
    id: str
    title: str
    status: ForeshadowStatus
    plant_chapter: str
    reveal_chapter: str | None
    description: str


def foreshadow_for_chapter(items: list[ForeshadowItem], chapter_id: str) -> list[ForeshadowItem]:
    return [
        item
        for item in items
        if item.status in {"潜伏", "触发"}
        and (item.plant_chapter == chapter_id or item.reveal_chapter == chapter_id)
    ]


def recover_foreshadow(item: ForeshadowItem, chapter_id: str) -> ForeshadowItem:
    if item.reveal_chapter != chapter_id:
        raise ValueError(f"{item.id} 不应在 {chapter_id} 回收")
    item.status = "已回收"
    return item
```

## 核心依赖版最小代码

```python
from pathlib import Path
from typing import Literal

import yaml
from pydantic import BaseModel, Field


class ForeshadowRecord(BaseModel):
    id: str
    title: str
    status: Literal["潜伏", "触发", "已回收", "弃用"] = "潜伏"
    plant_chapter: str | None = None
    reveal_chapter: str | None = None
    related_personas: list[str] = Field(default_factory=list)
    description: str


def load_foreshadows(path: Path) -> list[ForeshadowRecord]:
    if not path.exists():
        return []
    raw = yaml.safe_load(path.read_text(encoding="utf-8")) or []
    return [ForeshadowRecord.model_validate(item) for item in raw]


def save_foreshadows(path: Path, items: list[ForeshadowRecord]) -> None:
    data = [item.model_dump() for item in items]
    path.write_text(yaml.safe_dump(data, allow_unicode=True, sort_keys=False), encoding="utf-8")
```

## 落地建议

- 伏笔更新走 `foreshadow_update` 工具,不要让 LLM 直接编辑整份 Markdown。
- ID 生成由程序控制,避免模型生成重复编号。
- `伏笔/伏笔库.md` 可由结构化数据渲染,后续可增加 JSON sidecar。
- `/审核` 报告中列出本章应处理但未处理的伏笔。

## 验收标准

- 新建伏笔返回唯一 ID。
- 写到回收章节时,该伏笔完整条目会进入上下文。
- 已回收伏笔不会再次被当成潜伏伏笔注入。
- `/跨审` 能报告指定章节范围内的伏笔埋设、触发和回收状态。
