> / `CommandCheck` / `SkillRegistryView` / `SkillDirective.requires_states` /

## 业务背景

Writer Agent 的 MVP 分三阶段推进:阶段一 `/init → /目录`,阶段二 `/创意`/`/伏笔`/`/大纲`/`/骨架`,阶段三 `/创作 → /审核`。不同阶段允许的命令不同,例如未生成骨架前不能 `/创作`,正文未产出前不能 `/审核 正文`。

## 技术难点

难点在于 CLI 不是一次性命令工具,而是长生命周期的写作工作台。命令必须根据项目状态、标志文件、章节草稿和用户输入动态判断可用性。如果状态判断分散在各命令中,后续新增 `/修订`、`/跨审` 时很容易出现规则冲突。

## 解决方案

建立统一的 `ProjectState` 状态机,以 `AGENT.md` 为唯一状态总览,结合必要的文件存在性校验推导状态:

- S0 待启动:无 `AGENT.md`
- S1 初始化:已有 `AGENT.md`
- S2 大纲拟定中:已有 `outline/大纲.md`(或 `大纲/大纲.md`)
- S3 框架搭建中:已有 `outline/toc.md`(或 `目录/目录.md`)
- S4 正文编辑中:存在 `manuscript/*.md` / `正文草稿/*.md` / `正文/*.md`
- S5 审核中:所有章节定稿

命令注册表中声明每个命令的可用状态、条件谓词和错误提示模板。CLI 层只负责解析命令,执行前统一调用状态机校验。

## 最小 demo / 伪代码

```python
from enum import StrEnum
from pathlib import Path


class ProjectState(StrEnum):
    UNINITIALIZED = "S0"
    INITIALIZED = "S1"
    HAS_OUTLINE = "S2"
    HAS_TOC = "S3"
    WRITING = "S4"
    FINISHED = "S5"


# 历史设计（已删除，仅留档）：`COMMAND_ALLOWED` 静态表 + directive 端
# `DirectiveRegistry.state_matrix()` 曾共同驱动 `validate_command_available()`
# 做命令拦截。chg-remove-state-machine-enforcement 后这些符号全部移除，
# 状态机只保留展示用途，不再有拦截路径。


def detect_state(root: Path) -> ProjectState:
    """实际实现见 `src/writer/project/state.py::detect_state`,接受 Path | None。
    S0 路径(None root)→ UNINITIALIZED。**保留**:服务 `/状态` 展示。"""
    if not (root / "AGENT.md").exists():
        return ProjectState.UNINITIALIZED
    if any((root / "manuscript").glob("*.md")) \
       or any((root / "正文草稿").glob("*.md")) \
       or any((root / "正文").glob("*.md")):
        return ProjectState.WRITING
    if (root / "outline" / "toc.md").exists() or (root / "目录" / "目录.md").exists():
        return ProjectState.HAS_TOC
    if (root / "outline" / "大纲.md").exists() or (root / "大纲" / "大纲.md").exists():
        return ProjectState.HAS_OUTLINE
    return ProjectState.INITIALIZED
```

## 落地建议（现状）

- `writer.project.state` 只保留展示层：`detect_state(project_root)` /
  `inspect_project()` / `STATE_DESCRIPTIONS` / `ProjectSnapshot`。
- 不再有 `CommandSpec` / `COMMAND_ALLOWED` / `validate_command_available()`——
  命令入口不经过状态拦截。
- `AGENT.md` 的「当前状态」由 `render_agent_file` / `refresh_agent_file` 写入，
  只作 `/状态` 显示数据源，对 LLM 无硬墙含义。

## 验收标准（现状）

- `/状态` 在任意 S0-S5 下正确显示当前状态标签。
- `/大纲` / `/目录` 在任意状态（含 S4 正文编辑中）都能进入 directive body，
  由 LLM 根据实际文件判断追加 / 覆盖 / 查看。
- S0 用户误打 `/创作` 不被拦截，工具层兜底返回 `no_project_root`。
