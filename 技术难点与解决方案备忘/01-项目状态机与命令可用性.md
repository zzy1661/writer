# 项目状态机与命令可用性

## 业务背景

Writer Agent 的 MVP 分三阶段推进:阶段一 `/init → /目录`,阶段二 `/创意`/`/伏笔`/`/大纲`/`/骨架`,阶段三 `/创作 → /审核`。不同阶段允许的命令不同,例如未生成骨架前不能 `/创作`,正文未产出前不能 `/审核 正文`。

## 技术难点

难点在于 CLI 不是一次性命令工具,而是长生命周期的写作工作台。命令必须根据项目状态、标志文件、章节草稿和用户输入动态判断可用性。如果状态判断分散在各命令中,后续新增 `/修订`、`/跨审` 时很容易出现规则冲突。

## 解决方案

建立统一的 `ProjectState` 状态机,以 `AGENT.md` 为唯一状态总览,结合必要的文件存在性校验推导状态:

- S0 待启动:无 `AGENT.md`
- S1 初始化:已有 `AGENT.md`
- S2 大纲拟定中:已有 `outline/大纲.md`(或 `大纲/大纲.md`)
- S3 框架搭建中:已有 `outline/toc.md`(或 `目录/目录.md`)
- S4 正文编辑中:存在 `manuscript/*.md` / `正文草稿/*.md` / `正文/*.md`
- S5 审核中:所有章节定稿

命令注册表中声明每个命令的可用状态、条件谓词和错误提示模板。CLI 层只负责解析命令,执行前统一调用状态机校验。

## 最小 demo / 伪代码

```python
from enum import StrEnum
from pathlib import Path


class ProjectState(StrEnum):
    UNINITIALIZED = "S0"
    INITIALIZED = "S1"
    HAS_OUTLINE = "S2"
    HAS_TOC = "S3"
    WRITING = "S4"
    FINISHED = "S5"


# 实际 `src/writer/project/state.py::COMMAND_ALLOWED`(2026-07-08 实测):
# - 只列出"非 directive"命令;directive 命令(/大纲 /目录)的可用性
#   由 DirectiveRegistry.state_matrix() 派生,加新 directive 自动接入
COMMAND_ALLOWED = {
    "/init": {ProjectState.UNINITIALIZED},
    "/创作": {ProjectState.HAS_TOC, ProjectState.WRITING},
    "/审核": {ProjectState.WRITING, ProjectState.FINISHED},
    "/字数统计": {
        ProjectState.INITIALIZED, ProjectState.HAS_OUTLINE,
        ProjectState.HAS_TOC, ProjectState.WRITING, ProjectState.FINISHED,
    },
}


def detect_state(root: Path) -> ProjectState:
    """实际实现见 `src/writer/project/state.py::detect_state`,接受 Path | None。
    S0 路径(None root)→ UNINITIALIZED。"""
    if not (root / "AGENT.md").exists():
        return ProjectState.UNINITIALIZED
    if any((root / "manuscript").glob("*.md")) \
       or any((root / "正文草稿").glob("*.md")) \
       or any((root / "正文").glob("*.md")):
        return ProjectState.WRITING
    if (root / "outline" / "toc.md").exists() or (root / "目录" / "目录.md").exists():
        return ProjectState.HAS_TOC
    if (root / "outline" / "大纲.md").exists() or (root / "大纲" / "大纲.md").exists():
        return ProjectState.HAS_OUTLINE
    return ProjectState.INITIALIZED


def validate_command(root: Path, command: str) -> None:
    """实际实现见 `src/writer/project/state.py::validate_command_available`。
    优先查 `skill_registry.state_matrix()`,fallback 到 COMMAND_ALLOWED。"""
    state = detect_state(root)
    allowed = COMMAND_ALLOWED.get(command, set())
    if state not in allowed:
        raise ValueError(f"{command} 当前不可用,项目状态为 {state}")
```

## 核心依赖版最小代码

```python
from pathlib import Path

import typer
from pydantic import BaseModel
from rich.console import Console


console = Console()
app = typer.Typer()


class CommandCheck(BaseModel):
    command: str
    state: str
    ok: bool
    reason: str = ""


def check_command(project_root: Path, command: str) -> CommandCheck:
    state = detect_state(project_root)
    ok = state in COMMAND_ALLOWED.get(command, set())
    return CommandCheck(
        command=command,
        state=state,
        ok=ok,
        reason="" if ok else f"{command} 在 {state} 状态不可用",
    )


@app.command()
def write(chapter: str = "") -> None:
    check = check_command(Path.cwd(), "/创作")
    if not check.ok:
        console.print(f"[red]{check.reason}[/red]")
        raise typer.Exit(code=1)
    console.print(f"[green]开始写作章节:{chapter or '下一章'}[/green]")
```

## 落地建议

- 新增 `writer.project.state` 模块,提供 `detect_state(project_root)`。
- 新增 `CommandSpec` 数据结构,字段包含命令名、别名、可用状态、条件、处理器。
- 所有命令入口先经过 `validate_command_available()`。
- `AGENT.md` 的「当前状态」「当前章节」「当前进度」由状态机写入,其他模块只读。

## 验收标准

- 每个命令在 S0-S5 下有明确结果:可执行、条件执行或给出可读错误。
- `/创作` 在无骨架时必须被拦截。
- `/大纲` 在已有大纲后必须要求 `rewrite`、`append` 或 `view`。
- 非历史题材调用 `/史实` 返回明确错误,不会加载历史顾问。
