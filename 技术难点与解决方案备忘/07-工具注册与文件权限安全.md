# 工具注册与文件权限安全

## 业务背景

Agent 需要读写项目文件、解析 Markdown、统计字数、更新伏笔、注册章节,未来还可能调用 shell 和 git。工具能力越强,越需要控制写入范围和危险操作。

## 技术难点

LLM 工具调用天然有不确定性。若允许模型直接传路径或执行命令,可能误写项目外文件、覆盖用户资料,甚至执行危险 shell。另一方面,过度确认会破坏 CLI 写作体验。

## 解决方案

工具按范围分组并设置默认权限:

- 文件/目录工具:默认开启,但只能操作 `project_root` 内路径。
- 语义 IO 工具:默认开启,包括 Markdown 解析、字数统计、一致性检查。
- 项目专用工具:默认开启,封装伏笔更新、章节注册、RAG 查询。
- `shell_exec`:默认关闭。
- `git_ops`:commit 可配置开启,push 必须确认。

所有路径先 `Path.resolve()`,再与 `project_root` 比较。项目外路径必须由 CLI 层二次确认。

## 最小 demo / 伪代码

```python
from pathlib import Path


class ToolDeniedError(Exception):
    pass


class ToolRuntime:
    def __init__(self, project_root: Path) -> None:
        self.project_root = project_root.resolve()

    def safe_path(self, path: str) -> Path:
        target = (self.project_root / path).resolve()
        if self.project_root not in [target, *target.parents]:
            raise ToolDeniedError(f"路径越界: {target}")
        return target

    def read_file(self, path: str) -> str:
        target = self.safe_path(path)
        content = target.read_text(encoding="utf-8")
        if len(content) > 50_000:
            return content[:50_000] + "\n\n[内容已截断,请分段读取]"
        return content

    def shell_exec(self, command: str) -> str:
        if not is_shell_enabled():
            raise ToolDeniedError("shell_exec 默认关闭")
        if "rm -rf" in command:
            raise ToolDeniedError("拒绝危险命令")
        return run_command(command)
```

## 核心依赖版最小代码

```python
from pathlib import Path

from langchain_core.tools import tool


PROJECT_ROOT = Path.cwd().resolve()


def ensure_project_path(path: str) -> Path:
    resolved = (PROJECT_ROOT / path).resolve()
    if PROJECT_ROOT not in [resolved, *resolved.parents]:
        raise PermissionError(f"工具路径越界: {resolved}")
    return resolved


@tool
def safe_read_file(path: str) -> str:
    """读取项目目录内的文件,超长内容会截断。"""
    target = ensure_project_path(path)
    content = target.read_text(encoding="utf-8")
    return content[:50_000]


@tool
def safe_append_file(path: str, content: str) -> str:
    """向项目目录内文件追加内容。"""
    target = ensure_project_path(path)
    target.parent.mkdir(parents=True, exist_ok=True)
    with target.open("a", encoding="utf-8") as file:
        file.write(content)
    return "ok"
```

## 落地建议

- 工具实现放在 `writer.tools`,通过 LangChain `@tool` 暴露给 LangGraph `ToolNode`。
- 工具层统一抛出 `ToolDeniedError`、`ToolNotFoundError`、`ToolOutputTooLargeError`。
- `read_file` 对超大文件自动截断,提示使用搜索或分段读取。
- shell 命令建立拒绝列表,如 `rm -rf`、`chmod 777`、`mkfs.*`。

## 验收标准

- 任意工具不能通过 `../` 写出项目目录。
- 默认配置下 LLM 不能执行 shell。
- 读超大文件不会把 50k 字以上内容直接塞进上下文。
- 用户拒绝项目外路径后,节点能收到明确拒绝信息并转人工或换路径。

## 2026-07-08 修订:IO 错误处理约定

实际 builtin Tool 实测(`src/writer/tools/builtin/`):

| 错误类型 | 处理 |
| --- | --- |
| `ToolDeniedError`(`safe_path` 越界) | 上抛 → engine `except ToolError` → `ErrorEvent + Done(aborted)` |
| `ToolNotADirectoryError`(`safe_list_dir` 传文件) | 上抛 → 同上 |
| `PermissionError` / `OSError`(读盘失败) | 转 `ToolResult(metadata.error="io")`,**不外溢** |
| `UnicodeDecodeError`(非 UTF-8 文件) | `safe` 跳过,继续处理下一个文件;`project_search` 同理 |
| `ForeshadowLedgerSchemaError`(ledger 格式错) | 转 `ToolResult(metadata.error="schema")`,**不外溢** |
| S0 sentinel `__no_project__` | `foreshadow_search` 返回 `ToolResult(metadata.error="no_project_root")`,**不读盘** |

约定:文件 IO 错误 + 数据 schema 错误**不**让 turn 中断,LLM 看到 `ToolResult.output + metadata.error` 后自己决定下一步。`ToolDeniedError` / `ToolNotADirectoryError` 等"协议违反"错误**应该**外溢,因为它们通常意味着调用方 bug 而不是运行时数据问题。

## 2026-07-08 修订:`shell_exec` / `git_ops` 不在 builtin Tool 清单里

旧文档提到的 `shell_exec`(默认关闭)+ `git_ops`(commit 可配置,push 必须确认)在当前 builtin Tool 清单(6 个)里**没有**——LLM 通过 `safe_read_file` / `safe_write_file` + `project_search` 完成文件操作,不需要 shell。Git 操作由用户在自己 shell 里手动执行。`ToolRuntime.shell_enabled` 字段保留(per 备忘 13 §3),但当前 builtin Tool 都不读它。
