# 角色卡模板：三重标签生成法

每个角色写一个 `人物/<角色名>.md`。正文按 **三重标签生成法**（三层 schema）组织：

```
# <角色名>
## 表层标签（对外伪装，别人最先看到的）  ← Tier 1
## 中层特质（私下本性，最亲近的人才看得到）  ← Tier 2
## 底层内核（原生根源，藏在最深处）  ← Tier 3
## 附加信息（可选，仅在描述里有相关线索时填写）  ← Optional
```

三层是 **recommended schema 不是固定字段表**——LLM 工具循环可按描述信息量自主增删三层内的子项。缺信息字段统一写 `未知`，**禁止臆造**（生日 / 修为 / 籍贯等具体事实尤其禁止）。

`主要人物.md` 是索引文件，每行 `- 角色名：一句话简介`，**不**与单角色文件合并。

## 三层语义边界

| 层 | 视角 | 谁看得到 | 推动力的层级 |
|---|---|---|---|
| **表层标签（对外伪装）** | 第一印象 / 公开场景 | 所有人 | 形象工程（让 TA 在故事里显得可信、有趣、有冲突） |
| **中层特质（私下本性）** | 私下场景 | 主角 / 配角 / 亲密关系 | 性格深度（让 TA 在私下场景里丰满，避免扁平） |
| **底层内核（原生根源）** | 内心独白 / 关键时刻 | TA 自己 / 极少数人 | 角色动机（让 TA 的选择有因果驱动力，撑住人物弧） |

写作建议：每层各挑 1-3 个子项重点写，宁缺勿凑；底层越少越要写出因果链（"因为 X 所以 Y"），中层允许保留矛盾（"对外温柔 vs 私下暴躁"），表层可以略夸张以制造戏剧反差。

## 11 个推荐字段填写指引

### Tier 1 · 表层标签（对外伪装）

| 子项 | 类型 | 填写规则 |
|---|---|---|
| `### 性格` | 关键词 + 段落 | 3-5 个核心词 + 一段不超过 80 字的展开；可与中层「不对外展露的脾气」做对照反差 |
| `### 外貌` | 段落 | 含身高 / 体型 / 面容 / 标志性细节（痣、疤、发型等）；信息不足时写 `未知` |
| `### 谈吐` | 关键词 + 段落 | 语气 / 口头禅 / 用词习惯 / 公开场合说话风格；反派可写「阴阳怪气 / 总带反问」 |
| `### 身份` | 短标签 | 职业 / 阵营 / 在故事里扮演的角色 / 公开头衔；如「退伍军人 / 贫民窟守卫 / 门派第四代首席弟子」 |
| `### 社会面具` | 段落 | 对外展示的形象 / 想让别人相信的样子 / 公开行为模式（如「永远微笑着的便利店店长，但背后藏了家暴的儿子」） |

### Tier 2 · 中层特质（私下本性）

| 子项 | 类型 | 填写规则 |
|---|---|---|
| `### 独处习惯` | 列表或段落 | 独处时的典型行为：发呆 / 写日记 / 听老歌 / 收集奇怪小物件 / 深夜反复擦地板等 |
| `### 隐秘喜好` | 列表 | 不让外人知道的爱好、迷恋、收藏、暗瘾等（与表层「身份」不一定一致：如官员私下收藏同志漫画） |
| `### 不对外展露的脾气` | 段落 | 私下阴沉面 / 压抑的情绪 / 深夜独处时才会显露的性格（如「对外温柔」的 TA 私下会因一句失言砸杯子） |

### Tier 3 · 底层内核（原生根源）

| 子项 | 类型 | 填写规则 |
|---|---|---|
| `### 童年创伤` | 段落或事件清单 | 幼年至少年期的关键伤害 / 未愈合的旧伤 / 致使性格成因的事件（不必写细，关键事件 + 一句话影响即可） |
| `### 执念` | 关键词 + 段落 | 驱动他一切的念头 / 哪怕粉身碎骨也要追求的东西 / 内在 engine（如「要让母亲看到自己出人头地」） |
| `### 价值观底线` | 列表或段落 | 绝对不踩的红线 / 宁死不让步的原则 / 让他与人决裂的边界（如「绝不杀孩子」） |

## 缺失字段处理

- **缺信息字段统一写 `未知`**，禁止臆造（生日 / 修为 / 籍贯 / 出身等具体事实尤其禁止）。
- 子项可整段省略（连 `### 性格` 标题都不写）—— 比保留 `### 性格\n未知` 更清爽；下游 `_build_canon_block` 整篇 rglob 读取不会因此找不到。
- 部分信息片段也可成卡，例如「张三 主角 25 岁 程序员」即可建卡：`## 表层标签` 至少写 `### 身份` 「主角」+ `## 附加信息` 的 `### 登场年龄` 25 岁；其余子项按需写或省略。
- 用户后续补充信息时走 update 路径：用 `safe_read_file` 读原文件，再用 `safe_edit_file` 仅修改对应 `###` 子标题 / `##` 一级段内容，禁止整段覆写。

## 附加信息（可选）

`## 附加信息` 段保留八项固定字段遗留契约的可选项，**仅在用户描述里有相关线索时填写**。可写子项：

| 子项 | 类型 | 填写规则 |
|---|---|---|
| `### 登场时间` | 章节锚点 | 用 `第 X 章 / 第 X 卷 Y 章` 而非具体日期；信息不足时写 `未知` |
| `### 登场年龄` | 数字 + 单位 | 仅数字 + `岁`（如 `25 岁`）；未披露时写 `未知` |
| `### 能力` | 列表 | 强项 / 特长 / 修为层级（玄幻题材分境界、历史题材分官职） |
| `### 标志物品` | 列表 | 随身物 / 武器 / 信物，3-5 项 |
| `### 关联人物关系` | 反向链接 | 格式 `- [[人物名]]：关系类型，说明`；关系未披露时写 `未知` |

### 关联人物关系反向链接

- 格式：`- [[人物名]]：关系类型，一段说明`
- 例：`- [[李四]]：师兄，入门引路人，在第二章退出江湖`
- 仅在该角色档案确实存在 `人物/<对方名>.md` 时建立链接；未建立的关联写「提及但尚未建档」并放在说明里。
- 反向链接便于人类阅读 + 下游消费（`_build_canon_block` 整篇 rglob 读取不需要 link 解析）。

## 三层写作工作流（推荐顺序）

LLM 工具循环读取用户描述后，按这个顺序提取信息：

1. **先扫一眼表层** —— 姓名、身份、职业、阵营、外貌等最快能从描述里抽出来的。
2. **再挖中层** —— 「他独处时会……」/「他私下会……」/「他不让人看到的一面是……」类型的语义在描述里通常带「其实」「不过」「背地里」「其实」等反差点提示。
3. **最后挖底层** —— 「他为什么执念这件事」通常要追溯到童年 / 原生家庭 / 价值观起源；若描述里没有，可省略该子项而非臆造。
4. **附加信息挑能用上的写** —— 描述里出现章节 ID、年龄数字、技能、随身物才写，否则不强求。
5. **三层之间的因果链** —— 表层与中层之间应有反差（公开形象 vs 私下脾气），中层与底层之间应有因果（私下脾气源于童年创伤）。让角色立体。

## 多角色与追加

- `人物/主要人物.md` 是索引，每行 `- 角色名：一句话简介`，LLM 在 update 路径下应保留用户既有排序，按重要性 / 阵营 / 角色弧顺位追加。
- 单文件 = 单角色，**禁止**把多个角色合并到同一 `人物/<X>.md`。
- `人物/` 目录脚手架（`workspace.py:128`）创建空索引 `主要人物.md = "# 主要人物\n\n"`，本 directive 只做**追加**而非重写。

## canon-block 消费方式

- `_build_canon_block`（`prompts/context.py:119-146`）通过 `for relative in ("大纲", "人物")` 配合 `_read_markdown_files`（`context.py:241`）rglob 自动读取 `人物/` 下所有 `.md`。
- 因此每条角色卡的内容会作为正典资料注入 `plan_chapter` / `draft_chapter` / `review_chapter` 三处工作流的 LLM prompt，**三层结构性比八项固定字段更能驱动章节一致性**——尤其底层内核进 prompt 后能稳定保持人物弧线。

## 反例（避免写法）

- 把多个角色合并到同一 `人物/双男主.md` —— 违反单文件单角色约束。
- 在 `登场时间` 写 `2024-03-15` —— 应锚到章节而非日历日期。
- `关联人物关系` 用纯文本 `「李四（师兄）」` 而非 `[[李四]]` —— 失去反向链接可视化。
- 性格段写一整章人物小传 —— 限制 80 字以内，更长内容放「背景故事」附加节（可选调点）。
- **三层结构相同** —— 表层写「温柔」，中层写「温柔」，底层写「从小就被教育要温柔」—— 失去三层意义；每层应有反差点。
- **`### XXX\n未知` 占位大法** —— 字段真的写不出就整段省略子项，写 `未知` 是为了避免「假装知道」，但与下游「字段缺失影响章节生成」相权衡：表层字段建议保留（写 `未知` 字面即可），中层 / 底层字段若信息真的没有可省略。
