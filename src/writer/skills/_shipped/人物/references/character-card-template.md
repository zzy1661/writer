# 角色卡模板

每个角色写一个 `人物/<角色名>.md`，八个固定字段（不含 `## 人物` 自身）用 `##` 二级标题分节。`主要人物.md` 是索引文件，每行 `- 角色名：一句话简介`，**不**与单角色文件合并。

## 八项字段填写指引

| 字段 | 类型 | 填写规则 |
|---|---|---|
| 登场时间 | 章节锚点 | 用 `第 X 章 / 第 X 卷 Y 章` 而非具体日期；信息不足时写 `未知` |
| 登场年龄 | 数字 | 仅数字 + `岁`（如 `25 岁`）；未披露时写 `未知` |
| 性格 | 关键词 + 段落 | 3-5 个核心词 + 一段不超过 80 字的展开 |
| 外貌 | 段落 | 含身高 / 体型 / 面容 / 标志性细节（痣、疤、发型等） |
| 能力 | 列表 | 强项 / 特长 / 修为层级（玄幻题材分境界、历史题材分官职） |
| 标志物品 | 列表 | 随身物 / 武器 / 信物，3-5 项 |
| 关联人物关系 | 反向链接 | 格式 `- [[人物名]]：关系类型，说明`；关系未披露时写 `未知` |

## 缺失字段处理

- 信息不足的字段统一写 `未知`，**禁止臆造**具体事实（生日 / 修为 / 籍贯等）。
- 部分信息片段也可成卡，例如「张三 主角 25 岁 程序员」即可建卡，其余字段填 `未知`。
- 用户后续补充信息时走 update 路径：用 `safe_read_file` 读原文件，再用 `safe_edit_file` 仅修改对应 `##` 小节内容。

## 关联人物关系反向链接

- 格式：`- [[人物名]]：关系类型，一段说明`
- 例：`- [[李四]]：师兄，入门引路人，在第二章退出江湖`
- 仅在该角色档案确实存在 `人物/<对方名>.md` 时建立链接；未建立的关联写「提及但尚未建档」并放在说明里。
- 反向链接便于人类阅读 + 下游消费（`_build_canon_block` 整篇 rglob 读取不需要 link 解析）。

## 多角色与追加

- `人物/主要人物.md` 是索引，每行 `- 角色名：一句话简介`，LLM 在 update 路径下应保留用户既有排序，按重要性 / 阵营 / 角色弧顺位追加。
- 单文件 = 单角色，**禁止**把多个角色合并到同一 `人物/<X>.md`。
- `人物/` 目录脚手架（`workspace.py:128`）创建空索引 `主要人物.md = "# 主要人物\n\n"`，本 directive 只做**追加**而非重写。

## canon-block 消费方式

- `_build_canon_block`（`prompts/context.py:119-146`）通过 `for relative in ("大纲", "人物")` 配合 `_read_markdown_files`（`context.py:241`）rglob 自动读取 `人物/` 下所有 `.md`。
- 因此每条角色卡的内容会作为正典资料注入 `plan_chapter` / `draft_chapter` / `review_chapter` 三处工作流的 LLM prompt，**字段缺失会影响后续章节生成**——保持每项至少 `未知` 占位比留空更稳定。

## 反例（避免写法）

- 把多个角色合并到同一 `人物/双男主.md` —— 违反单文件单角色约束。
- 在 `登场时间` 写 `2024-03-15` —— 应锚到章节而非日历日期。
- `关联人物关系` 用纯文本 `「李四（师兄）」` 而非 `[[李四]]` —— 失去反向链接可视化。
- 性格段写一整章人物小传 —— 限制 80 字以内，更长内容放「背景故事」附加节（可选调点）。
