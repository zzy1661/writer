# 整体架构方法参考（八法速选表）

本参考为 `/大纲` directive 在生成大纲时的**骨架选择**提供输入。`/大纲` 命中后调用本 reference 的方法按 `AGENT.md` 中 `架构方法:` 字段挑选对应骨架；缺省字段回退到 **雪花法**。本参考**不锁定方法词汇**，作者可以自创复合写法（例如「三幕+英雄之旅」「雪花+节拍表」）。

> 与 `docs/写作方法论/写作架构方法.md` 的关系：那是主文档（详尽叙述），本参考是给 LLM 工具循环选骨架用的速查表——大纲骨架、节拍数、典型章节模板三件套。

## 速选流程（先问题材，再问篇幅，再选方法）

| 题材 / 篇幅 | 推荐方法 | 理由 |
|---|---|---|
| 玄幻 长篇 | 雪花法 + 三步八段式 | 长篇需要多层细化，金手指+升级可走八段 |
| 玄幻 短篇 | 三幕结构 + 英雄之旅 | 短篇节奏紧凑，成长弧清晰 |
| 言情 长篇 | 人物弧光架构（雪花层 2 → 4） | 情感关系是核心驱动 |
| 言情 短篇 | 情感节拍表 | 9 段精确控糖虐比 |
| 历史 长篇 | 布莱克节拍表 + 大事年表 | 史料严苛，节奏要稳 |
| 历史 中篇 | 三幕结构 + 史实锚点 | 嵌套单次政权变动 |
| 都市 现实 长篇 | 三明治架构 | 主线 + 多角色支线 + 单元串联 |
| 校园 职业 | 单元串联架构 | 适合长期连载 |
| 悬疑 反转 | 环形叙事架构 | 首尾呼应，揭示开篇伏笔 |
| 短篇 新手 | 起承转合 / 两点一线法 | 极简单点，最易上手 |
| 反套路 | 反英雄 / 群像 / 开放式 框架 | 文学向探索 |

## 八大方法骨架（按推荐度排序）

### 1. 雪花法（Snowflake Method，**默认**）

六层由小到大：

1. 一句话核心故事（全书总纲）。
2. 四段式短梗概：开端 / 发展 / 高潮 / 结局。
3. 人物小传：性格 / 执念 / 弱点 / 目标 / 弧光 / 终极结局。
4. 章节长梗概：数十段，对应全书各大篇章与关键剧情节点（标注转折 / 低谷 / 爽点 / 伏笔）。
5. 细化场景与支线。
6. 完整成文。

**大纲输出骨架**：

```
# <书名>

## 一句话核心
<谁，经历了什么冲突，最终得到/失去了什么>

## 四段式短梗概
- 开端：...
- 发展：...
- 高潮：...
- 结局：...

## 主要人物小传
- <人物名>：<性格><执念><弱点><目标><弧光概述>

## 章节长梗概（按幕划分）
### 第一幕 / 建置
- 第 X 章：<标题> —— <核心剧情>，<转折/低谷/爽点>
- ...

### 第二幕 / 对抗
- ...

### 第三幕 / 转折
- ...

### 第四幕 / 解决
- ...
```

### 2. 三幕结构（Three-Act Structure）

三幕 / 四幕衍生（占全书 25% / 50% / 25%）：

- **第一幕·铺垫（25%）**：日常世界 → 触发事件 → 第一情节点。
- **第二幕·对抗（50%）**：主角进入新世界 → 障碍升级 → 中点反转 → 至暗时刻。
- **第三幕·结局（25%）**：终局决战 → 真相揭晓 → 新旧生活对比。

**大纲输出骨架**：

```
# <书名>

## 前提
<一句话前提>

## 三幕大纲
### 第一幕·铺垫（25%）
- 第 1-3 章：日常世界 + 主角人设
- 第 4-5 章：触发事件 + 第一情节点

### 第二幕·对抗（50%）
- 第 6-15 章：应对挑战，盟友与反派入场
- 第 15 章：中点反转
- 第 15-20 章：阻力升级

### 第三幕·结局（25%）
- 第 20-22 章：至暗时刻 + 核心真相
- 第 23-25 章：终局 + 新平衡
```

### 3. 英雄之旅（Hero's Journey）

12 阶段：

1. 平凡世界 → 2. 召唤冒险 → 3. 拒绝召唤 → 4. 遇见导师 → 5. 跨越门槛 → 6. 试炼之路 → 7. 接近洞穴 → 8. 终极考验 → 9. 获取奖励 → 10. 归途重生 → 11. 携赐回归

（11 与 12 阶段在不同版本的归纳里略有差异，本骨架按坎贝尔原版 12 段落到 11/12 合并为「归途/回归」。）

**大纲输出骨架**：

```
# <书名>

## 英雄之旅（12 阶段）
1. 平凡世界：<第 X 章> —— <主角日常 + 缺失>
2. 召唤冒险：<第 Y 章> —— <触发事件，召唤来临>
3. 拒绝召唤：<第 Z 章> —— <主角迟疑 / 退缩>
4. 遇见导师：<...> —— <引路人出现>
5. 跨越门槛：<...> —— <决定踏上旅程>
6. 试炼之路：<...> —— <盟友与早期试炼>
7. 接近洞穴：<...> —— <核心挑战临近>
8. 终极考验：<...> —— <第一情节点>
9. 获取奖励：<...> —— <中点反转>
10. 归途重生：<...> —— <阻力升级>
11. 携赐回归：<...> —— <终局 + 新平衡>
```

### 4. 三步八段式（网文升级流专用）

8 个章节段位：开端 → 转机 → 成长 → 冲突 → 低谷 → 蜕变 → 决战 → 收尾。

**大纲输出骨架**：

```
# <书名>

## 八段式大纲
1. 开端（铺垫 + 主角困境）：<...>
2. 转机（金手指/机遇）：<...>
3. 成长（第一次小崛起）：<...>
4. 冲突（强敌登场）：<...>
5. 低谷（主角惨败）：<...>
6. 蜕变（闭关/顿悟）：<...>
7. 决战（核心反派）：<...>
8. 收尾（复仇/登顶 + 新篇）：<...>
```

### 5. 三明治架构（Sandwich Method）

顶层主线 + 中层人物/支线 + 底层章节细节。

**大纲输出骨架**：

```
# <书名>

## 顶层·主线大纲
- <幕 1>：<核心事件>
- <幕 2>：<核心事件>
- <幕 3>：<核心事件>

## 中层·人物弧光 + 支线
- <人物 A>：<弧光阶段 1> → <阶段 2> → <阶段 3>
- <人物 B>：<支线动机>
- <伏笔 X>：<铺垫位置> → <回收位置>

## 底层·章节细节（按主线展开）
- 第 N 章：<场景> / <对话锚点> / <视角> / <节奏>
```

### 6. 布莱克节拍表（Blake Snyder Beat Sheet）

15 个固定节拍：

1. 开篇画面（Opening Image）
2. 主题陈述（Theme Stated）
3. 铺垫（Setup）
4. 触发事件（Catalyst）
5. 辩论（Debate）
6. 第二幕入口（Break into Two）
7. B 故事（副线 / 感情线）
8. 乐趣与游戏（Fun and Games）
9. 中点（Midpoint）
10. 坏人逼近（Bad Guys Close In）
11. 一切失败（All Is Lost）
12. 灵魂的暗夜（Dark Night of the Soul）
13. 第三幕入口（Break into Three）
14. 终局（Finale）
15. 结局画面（Final Image）

**大纲输出骨架**：

```
# <书名>

## 节拍表（15 节拍）
| # | 节拍 | 章节锚点 | 内容 |
|---|---|---|---|
| 1 | 开篇画面 | 第 1 章 | <...> |
| 2 | 主题陈述 | 第 X 章 | <...> |
| 3 | 铺垫 | <...> | <...> |
| ... | ... | ... | ... |
| 15 | 结局画面 | 第 N 章 | <...> |
```

### 7. 人物弧光架构（Character Arc）

以主角**内心变化**为主架，不以事件为主。

6 步：初始性格 → 外部冲击 → 内心挣扎 → 认知崩塌 → 重塑自我 → 最终人格。

**大纲输出骨架**：

```
# <书名>

## 主角弧光
1. 初始性格：<核心特征 + 内在缺失>
2. 外部冲击：<触发事件>
3. 内心挣扎：<防御机制失败>
4. 认知崩塌：<旧世界观崩解>
5. 重塑自我：<新认知形成>
6. 最终人格：<弧光落点>

## 关键关系节点（与弧光对齐）
- <配角 A>：在阶段 N 推动主角前进
- ...
```

### 8. 起承转合（中式四步，最简）

```
# <书名>

## 起承转合
- 起（引入人物背景）：<...>
- 承（矛盾逐步发展）：<...>
- 转（重大转折/危机）：<...>
- 合（尘埃落定）：<...>
```

## 多方法混合（场景切换）

- **玄幻长篇**：雪花法（顶层）+ 三步八段式（每卷中观节拍）+ 节拍表（卷末高潮）。
- **言情长篇**：人物弧光架构（主角）+ 情感节拍表（CP 关系）+ 单元串联（每卷独立小故事）。
- **历史中篇**：三幕结构（全书）+ 大事年表（史实对齐）+ 节拍表（次要情节点定位）。

## 写作技巧与禁忌

- **方法服务于故事，不为方法服务**：先确认核心冲突类型（情感 / 战斗 / 解谜 / 成长）再选方法。
- **复合使用方法时主次分明**：顶层用 1-2 个方法，细节层可补充；不要每章都换方法骨架。
- **节拍数量与篇幅对齐**：20 万字以上用 4 幕 16 章 × 7 卷 = 100+ 章结构；短篇不要硬套雪花 6 层，2-3 层即可。
- **避免方法漂移**：选定方法后中途切换会让读者感到"读了两本书"，尤其跨方法时。

## 大纲字段速查（按方法补充）

| 方法 | 大纲必含字段 |
|---|---|
| 雪花法 | 一句话核心 / 四段式 / 人物小传 / 章节长梗概 |
| 三幕结构 | 前提 / 三幕比例 / 第一情节点 / 中点 / 至暗时刻 |
| 英雄之旅 | 12 阶段锚点 / 导师设定 / 跨越门槛心理转折 |
| 三步八段式 | 8 段位 / 升级点 / 爽点密度 |
| 三明治 | 主线 / 人物弧 / 章节细节 |
| 布莱克节拍表 | 15 节拍对照 / 副线 / 至暗时刻 |
| 人物弧光 | 主角内心阶段 / 关键关系节点 / 弧光落点 |
| 起承转合 | 4 步简短大纲 |

## 反例（避免写法）

- 玄幻长篇用起承转合（4 步太短，无法承载百万字）。
- 言情短篇用雪花法 6 层（信息密度过大，不如直接列情感节拍）。
- 历史题材无视史实锚点（即便用三幕/雪花，史实对齐仍是硬约束）。
- 短篇强行铺「升级表」（玄幻套路在短篇里会拖垮节奏）。
- 全书大杂烩：每幕换方法，读者读感割裂。
