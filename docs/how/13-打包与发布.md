# 13 · 打包与发布(PyInstaller)

> 对应代码:`writer.spec` + `src/writer/__main__.py` + `packaging/`(如存在)

## 13.1 设计动机

**问题**:用户可能是非 Python 开发者,不应该要求他们 `uv sync + 配置 env`。

**方案**:**PyInstaller** 把 `writer-agent` 打包成单文件可执行:

- macOS / Linux:`dist/writer`
- Windows:`dist/writer.exe`

## 13.2 PyInstaller 配置 `writer.spec`

```python
# writer.spec (简化版)
# -*- mode: python ; coding: utf-8 -*-

a = Analysis(
    ["src/writer/cli/main.py"],
    pathex=[],
    binaries=[],
    datas=[
        # Markdown 资源(SKILL.md / agent .md / 模板)
        ("src/writer/skills/_shipped", "writer/skills/_shipped"),
        ("src/writer/agents/_shipped", "writer/agents/_shipped"),
        ("src/writer/prompts", "writer/prompts"),
    ],
    hiddenimports=[
        "writer.engine",
        "writer.routing",
        # ... 所有包,避免 lazy import 漏掉
    ],
    hookspath=[],
    runtime_hooks=[],
    excludes=[
        "pytest", "mypy", "ruff",
        "tkinter", "unittest",
    ],
    noarchive=False,
)
pyz = PYZ(a.pure)

exe = EXE(
    pyz,
    a.scripts,
    a.binaries,
    a.datas,
    [],
    name="writer",
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=False,  # 压缩
    console=True,
    icon=None,
)
```

### 关键配置

- **`datas`**:必须包含 Markdown 资源目录(SKILL.md / agent .md / prompts/),PyInstaller 默认不打非 `.py` 文件
- **`hiddenimports`**:列出所有 lazy import 的包,避免运行时 `ModuleNotFoundError`
- **`excludes`**:排除 dev-only 依赖(pytest / mypy / ruff)与 GUI 库(tkinter)
- **`console=True`**:CLI 工具保留终端窗口

## 13.3 构建命令

```bash
# 1. 同步开发依赖(包含 PyInstaller)
uv sync --all-extras

# 2. 在目标系统上构建(macOS / Windows 各自构建)
PYINSTALLER_CONFIG_DIR=tmp/pyinstaller \
  uv run pyinstaller writer.spec

# 3. 产物位于 dist/
dist/writer      # macOS / Linux
dist/writer.exe  # Windows
```

### 跨平台构建

> PyInstaller 不支持交叉编译,macOS 与 Windows 必须各自在原生系统构建。

CI 建议:

```yaml
# .github/workflows/release.yml
jobs:
  build-macos:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - uses: astral-sh/setup-uv@v1
      - run: uv sync --all-extras
      - run: PYINSTALLER_CONFIG_DIR=tmp/pyinstaller uv run pyinstaller writer.spec
      - uses: actions/upload-artifact@v3
        with:
          name: writer-macos
          path: dist/writer

  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - uses: astral-sh/setup-uv@v1
      - run: uv sync --all-extras
      - run: pyinstaller writer.spec
      - uses: actions/upload-artifact@v3
        with:
          name: writer-windows
          path: dist/writer.exe
```

## 13.4 资源路径处理

打包后,Markdown 资源通过 `__file__` 解析:

```python
# src/writer/skills/directive_discovery.py
from pathlib import Path

_SHIPPED_DIR = Path(__file__).parent / "_shipped"


def discover_shipped_directives() -> list[SkillDirective]:
    if not _SHIPPED_DIR.exists():
        return []
    return [_parse_skill_md(p / "SKILL.md") for p in _SHIPPED_DIR.iterdir() if (p / "SKILL.md").exists()]
```

**PyInstaller 解包后**:`Path(__file__)` 指向 `sys._MEIPASS/writer/skills/_shipped`,正常工作。

### 验证资源被正确打包

```bash
# macOS / Linux
./dist/writer/writer doctor
# 应输出 Model / Base URL / API Key 配置信息,无 ModuleNotFoundError

# Windows
dist\writer\writer.exe doctor
```

## 13.5 故障排查

### 常见问题

| 现象 | 原因 | 解决 |
| ---- | ---- | ---- |
| `ModuleNotFoundError: writer.engine` | lazy import 包未列在 `hiddenimports` | 加进 `hiddenimports` |
| `[Errno 2] No such file or directory: '_shipped/大纲/SKILL.md'` | Markdown 资源未打包 | 加进 `datas` |
| `REPL 启动 FileNotFoundError` | 同上,或资源路径用了 `__file__` 之外的相对路径 | 改用 `Path(__file__).parent / "_shipped"` |
| 打包后体积 > 100MB | 包含不必要的 ML / GUI 库 | 加进 `excludes` |

### 验证 Markdown 资源

```python
# 在打包后的可执行里运行 Python 探针
import writer.skills.directive_discovery as d
print(d.discover_shipped_directives())
# 应列出 [大纲, 目录]
```

## 13.6 用户体验流程

```
1. 用户下载 dist/writer(macOS)或 dist/writer.exe(Windows)

2. 首次运行:
   $ ./writer doctor
   Model:    gpt-4o-mini
   Base URL: https://api.openai.com/v1
   API Key:  未配置

3. 配置 API Key(项目级):
   $ ./writer new 长安程序员
   $ cd 长安程序员
   $ vim .writer/config   # 填入 WRITER_API_KEY
   $ ./writer
   writer> /大纲 穿越到唐朝
   ...
```

### 注意事项

- **macOS 首次运行** 可能被 Gatekeeper 拦截 → 「系统设置 → 隐私与安全性 → 仍要打开」
- **Windows Defender** 可能误报 → 添加白名单
- **修改源码后必须重新构建** 才能生效
