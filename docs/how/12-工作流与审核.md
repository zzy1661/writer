## 12.1 设计动机

**问题**：LLM 工具循环擅长**短交互**（读文件 → 写文件 → 回答），但**长任务**（写一章 3000 字、跨章一致性检查）需要**多阶段编排**——

- 阶段 1：基于大纲 + 上一章摘要生成 outline
- 阶段 2：写正文（可分多段，每段都校验连续性）
- 阶段 3：proofread（语法 / 错别字）
- 阶段 4：review_gate（结构化评分，决定是否回流）
- 阶段 5：persist_outputs（原子写文件 + 更新 `chapter_summaries.json`）

**两种编排范式**：

| 范式                | 状态机                       | 优势                                  | 劣势                       |
| ------------------- | ---------------------------- | ------------------------------------- | -------------------------- |
| 手写同步 stub       | 自由函数                     | 简单、可控                            | 真实长任务还要写 LangGraph |
| **LangGraph `StateGraph`**（本项目当前） | 节点 + 边 + checkpointer     | 自动 checkpoint / interrupt / resume | 学习成本稍高，需 state schema |

**当前选择**（per `real-writing-pipeline` PR2 + PR3）：LangGraph `StateGraph`，checkpointer 默认 `MemorySaver`，有 `project_root` 时降级到 `SqliteSaver`；引擎在 `_run_workflow` 中将 `result.status="completed"` 映射到 `Done(reason="workflow_completed", payload={artifacts, metrics})`。

## 12.2 `WorkflowResult` —— 工作流输出契约

> 对应代码：`src/writer/workflows/types.py`

```python
@dataclass(frozen=True)
class WorkflowResult:
    status: Literal["completed", "pending", "failed"]
    chunks: list[str] = field(default_factory=list)        # 流式文本块
    artifacts: dict[str, Path] = field(default_factory=dict)  # 产物路径
    metrics: dict[str, Any] = field(default_factory=dict)  # 决策指标


class WorkflowStub(Protocol):
    """工作流签名：接收 RunnerContext + RunnerDeps，返回 WorkflowResult。"""
    def __call__(self, ctx: RunnerContext, deps: RunnerDeps) -> WorkflowResult: ...
```

### `status` 语义（per `engine.py::_run_workflow`）

| status      | Done reason        | payload 关键字段                                    | 含义                                                |
| ----------- | ------------------ | --------------------------------------------------- | --------------------------------------------------- |
| `completed` | `workflow_completed` | `artifacts` / `metrics`                            | 成功跑完                                            |
| `pending`   | `aborted`          | `decision="needs_rewrite"`                          | 需要上游动作（PR3+；当前 write_chapter 无此分支）    |
| `failed`    | `aborted`          | `error`                                             | 真实失败                                            |

**`pending` 语义**：当工作流产出一个「需要上游动作」的信号（例如 review_chapter 的 `needs_rewrite`），引擎以 `aborted` 暴露并附 `decision` metric，让 REPL 区分「需要重写」与「失败」。

```python
from writer.workflows.review_chapter import run as review_chapter_run
from writer.workflows.write_chapter import run as write_chapter_run

# 工作流 callable 接受 (ctx, deps) 并返回 WorkflowResult。
# 旧 2-arg 形态 (Callable[[RunnerContext], ...]) 仍被接受以兼容插件。
WorkflowStub = Callable[..., "WorkflowResult | Iterable[str]"]

WORKFLOWS: dict[str, WorkflowStub] = {
    "write_chapter": write_chapter_run,
    "review_chapter": review_chapter_run,
}


def run_workflow(name: str, ctx: RunnerContext, deps: RunnerDeps) -> WorkflowResult:
    """RunnerDeps.run_workflow 的辅助函数：查找 WORKFLOWS，执行对应实现。"""
    stub = WORKFLOWS.get(name)
    if stub is None:
        raise WorkflowNotFoundError(f"未知工作流 {name!r}; available: {sorted(WORKFLOWS)}")
    return stub(ctx, deps)
```

### `_DefaultRunnerDeps.run_workflow`

```python
def run_workflow(self, name: str, ctx: RunnerContext) -> WorkflowResult:
    runner = self._workflows.get(name)
    if runner is None:
        available = sorted(self._workflows)
        raise WorkflowNotFoundError(
            f"未知工作流 {name!r}; available: {available}"
        )
    from writer.workflows import run_workflow as _dispatch
    return _dispatch(name, ctx, self)
```

## 12.5 `write_chapter` —— LangGraph 5 节点图（PR2 实装）

> 对应代码：`src/writer/workflows/write_chapter.py`

`write_chapter.run(ctx, deps)` 构建并跑完 LangGraph 图，返回 `WorkflowResult`。下面是图的拓扑（来自 `build_writer_graph`）：

```
prep_context → plan_chapter → draft_chapter → proofread
                                          ↓
                                    review_gate
                                    /        \
                              (rewrite)    (end)
                                  ↓           ↓
                            draft_chapter   persist_outputs → END
```

### 节点职责

| 节点              | 职责                                                            | 是否调用 LLM         |
| ----------------- | --------------------------------------------------------------- | -------------------- |
| `prep_context`    | 调用 `writer.prompts.context.prep_context` 组装 canon block     | 否                   |
| `plan_chapter`    | **LLM 驱动**自由散文式计划(调 `_call_plan_chapter` + `prompts.agents.CHAPTER_PLAN_TEMPLATE`);`deterministic` 模式 strict-reject(`raise RuntimeError` 要求 `WRITER_API_KEY`) | **是(强制 real)**   |
| `draft_chapter`   | 调 `deps.prose_client.generate_text` 写正文                       | 视 prose_client 而定 |
| `proofread`       | 轻量 lint（短草稿告警）                                          | 否                   |
| `review_gate`     | deterministic 路径 score 8 自动通过；real 路径调 LLM 结构化 ReviewVerdict | 是（仅 real 路径）   |
| `persist_outputs` | 写 `草稿/chapter-<chapter_id>.md` + 原子更新 `chapter_summaries.json` | 否                   |

### checkpointer

```python
def _build_checkpointer(project_root: Path | None) -> tuple[Any, Callable[[], None]]:
    try:
        from langgraph.checkpoint.sqlite import SqliteSaver
        if project_root is None:
            raise ImportError
        checkpoint_dir = project_root / ".writer"
        checkpoint_dir.mkdir(parents=True, exist_ok=True)
        conn = sqlite3.connect(
            str(checkpoint_dir / "checkpoints.sqlite"),
            check_same_thread=False,
        )
        return SqliteSaver(conn), conn.close
    except ImportError:
        from langgraph.checkpoint.memory import MemorySaver
        return MemorySaver(), lambda: None
```

**两层降级**：有 `project_root` 且 `SqliteSaver` 可用 → `.writer/checkpoints.sqlite`；否则 `MemorySaver`。`thread_id` 用 `ctx.session_id` 或 `f"write-{chapter_id}"`，让同一 chapter_id 跨 REPL 轮次可恢复。

### `ReviewVerdict`（结构化审核输出）

```python
class ReviewVerdict(BaseModel):
    pass_: bool = Field(alias="pass")
    score: int = Field(ge=0, le=10)
    concerns: list[str] = Field(default_factory=list)
```

`REVIEW_THRESHOLD = 7` 是通过门；`max_retries = 2` 让循环最多跑 `max_retries + 1` 次（首次 + 两次回流）。

### 引擎依赖注入（节点级）

LangGraph 节点是接受 `state` 并返回部分状态的裸函数，无法直接传 `deps`。`write_chapter.run` 在每次 `graph.invoke` 前 `_set_deps(deps)`、之后 `_reset_deps()`，把 `deps` 通过模块级 `_WORKFLOW_DEPS` 注入节点。这是 LangGraph 文档中跑 scoped state 的官方模式。

### `prose_client` 分支

```python
def _require_prose_client(deps: RunnerDeps) -> Any:
    client = deps.prose_client
    if client is None:
        msg = "RunnerDeps.prose_client is None; production_deps always sets it"
        raise RuntimeError(msg)
    return client
```

`production_deps()` 总是装配一个 `prose_client`（有 API key 时 `RealProseClient`，否则 `DeterministicProseClient`）。`review_gate` 根据 `deps.prose_client.name` 分支：

- `"deterministic"` → 永远以 score 8 通过，无 LLM 调用
- `"real"` → 调 `_call_review_llm`（用 `deps.review_llm` 或 `get_llm(settings)`）走 `ReviewVerdict` 结构化输出

## 12.6 `review_chapter` —— LangGraph 5 节点图（PR3 实装）

> 对应代码：`src/writer/workflows/review_chapter.py`

`review_chapter.run(ctx, deps)` 构建并跑完 LangGraph 图，返回 `WorkflowResult`。图拓扑（来自 `build_reviewer_graph()`）：

```
load_target_chapter → prep_review_context → aggregate_reviews → decision_gate → persist_review_report → END
```

### 节点职责

| 节点                      | 职责                                                                                                          | 是否调用 LLM         |
| ------------------------- | ------------------------------------------------------------------------------------------------------------- | -------------------- |
| `load_target_chapter`     | 解析 `target`（`"current"` 取最新章节，否则按 `chapter-<id>.md`），加载章节正文；项目根未绑定时硬失败 | 否                   |
| `prep_review_context`     | 调 `deps.tool_registry.invoke("foreshadow_search", status="active")` 加载活跃伏笔 IDs；失败非致命（空列表继续）| 否                   |
| `aggregate_reviews`       | 单次 LLM 调用产出 `MultiConcernReview`（continuity / pacing / prose 三 concern + total_score + summary）       | 视 prose_client 而定 |
| `decision_gate`           | 把 `total_score` + 每 concern pass 标志映射为 `"pass"` / `"tweak"` / `"needs_rewrite"`                         | 否                   |
| `persist_review_report`   | 写 `草稿/reviews/chapter-<id>-<ISO-timestamp>.json`，无项目根时跳过持久化但仍返回带决策的 `WorkflowResult`   | 否                   |

### `ReviewerState`（TypedDict）

```python
class ReviewerState(TypedDict, total=False):
    target: str                        # "current" 或具体 chapter_id
    focus: list[str]                   # 用户传入的关注点列表
    chapter_id: str
    chapter_path: str
    chapter_text: str
    active_foreshadows: list[str]
    review: dict[str, Any]             # MultiConcernReview 转 dict
    decision: str                      # "pass" | "tweak" | "needs_rewrite"
    artifacts: dict[str, str]
    metrics: dict[str, float | int | str]
    trace: list[str]
```

### `MultiConcernReview`（结构化审核输出）

```python
class ConcernVerdict(BaseModel):
    score: int = Field(ge=0, le=10)
    pass_: bool = Field(alias="pass")
    findings: list[str] = Field(default_factory=list)


class MultiConcernReview(BaseModel):
    continuity: ConcernVerdict
    pacing: ConcernVerdict
    prose: ConcernVerdict
    total_score: int
    summary: str
```

### 决策门（`decision_gate` 映射规则）

| 条件 | decision |
| ---- | -------- |
| `total_score >= 8` **AND** 三个 concern 全部 `pass_` | `"pass"` |
| `total_score < 6` **OR** 任一 concern `score < 4` | `"needs_rewrite"` |
| 其他（4 ≤ total_score < 8，或某 concern score 偏低） | `"tweak"` |

阈值常量定义于 `src/writer/workflows/types.py`：`DECISION_PASS_THRESHOLD = 8`、`DECISION_TWEAK_THRESHOLD = 6`、`DECISION_NEEDS_REWRITE_CONCERN = 4`。

### status ↔ decision 映射

`pending` = "needs_rewrite"，引擎以 `Done(aborted, payload={"decision": "needs_rewrite"})` 暴露，让 REPL 区分「需要重写」与「失败」。

### `prose_client` 分支

`_aggregate_reviews_node` 内部调 `_llm_review`，再 fallback `_deterministic_review`：

- `review_llm` 注入 → 直接用 fake / 测试 LLM（测试路径）
- `review_llm is None` + `settings.has_api_key=True` → 调 `get_llm(settings)` 走真实 LLM（生产路径）
- 无 API key / LLM 调用失败 → 降级到 `ConcernVerdict(score=4, pass=False)` + findings 包含错误信息（per `ad45896` Bug 03 修复）

### 引擎依赖注入（节点级）

与 `write_chapter` 同模式：节点是裸函数，无法直接传 `deps`；`review_chapter.run` 在每次 `graph.invoke` 前 `_set_deps(deps)`、之后 `_reset_deps()`，把 `deps` 通过模块级 `_REVIEW_DEPS` 注入节点。

> **Bug 03 修复历史**（per `ad45896`）：之前 `_aggregate_reviews_node` 早返 `_deterministic_review` 路径，即使 `WRITER_API_KEY` 已配也走确定性；现在 fallback 推迟到 `_llm_review` 内部 `get_llm(settings)` 自然走 API key，让测试注入的 fake 与生产 API key 都稳定生效。

## 12.7 引擎 `_run_workflow` 分派

> 对应代码：`src/writer/engine/engine.py::Engine._run_workflow`

```python
async def _run_workflow(self, name: str, ctx: RunnerContext) -> AsyncIterator[TextChunk | Done]:
    deps = self._deps
    cfg = self._cfg

    if not cfg.fast_mode:
        yield TextChunk(text=f"[engine] 工作流 {name} 启动\n")
    result = deps.run_workflow(name, ctx)
    for chunk in result.chunks:
        yield TextChunk(text=chunk)
    if result.status == "completed":
        yield Done(
            reason="workflow_completed",
            payload={
                "workflow": name,
                "artifacts": {k: str(v) for k, v in result.artifacts.items()},
                "metrics": dict(result.metrics),
            },
        )
        return
    if result.status == "pending":
        decision = str(result.metrics.get("decision", "needs_rewrite"))
        yield Done(
            reason="aborted",
            payload={
                "workflow": name,
                "decision": decision,
                "artifacts": {k: str(v) for k, v in result.artifacts.items()},
                "metrics": dict(result.metrics),
            },
        )
        return
    # status == "failed"
    error_msg = str(result.metrics.get("error", "")) or f"工作流 {name} 失败"
    yield Done(reason="aborted", payload={"workflow": name, "error": error_msg})
```

**`workflow_pending` 已删除**：它不再是合法的 `DoneReason`；pending 分支走 `aborted + decision` 表达。

### `decision` metric 透传

`pending` 状态以 `Done(reason="aborted", payload={"decision": "needs_rewrite"})` 暴露。REPL 渲染时：

```python
if done.reason == "aborted" and "decision" in done.payload:
    console.print(f"[yellow]工作流 {done.payload['workflow']} 决策: {done.payload['decision']}[/yellow]")
```

让用户知道「需要重写」而非「失败」。

## 12.8 完整数据流：`/创作 1.1 第一章`

```
CLI 收到 "/创作 1.1 第一章"
   ↓
REPL: handle_repl_input → session.run_turn(text)
   ↓
Engine.run_turn:
    refresh_project_state() → project_state == "S1" (INITIALIZED)
    ctx = RunnerContext(user_input="/创作 1.1 第一章",
                         project_root=..., project_state="S1", session_id=...)
    session.engine.run(ctx)        # Engine.run 是主入口
        ↓
Engine._engine_loop:
    deps.route(...) → AgentAction(start_workflow, workflow="write_chapter")
    match action.action_type:
        case "start_workflow":
            async for event in _run_workflow("write_chapter", ctx, deps, cfg):
                yield event
   ↓
_run_workflow:
    yield TextChunk("[engine] 工作流 write_chapter 启动\n")
    result = deps.run_workflow("write_chapter", ctx)
        → workflows/run_workflow("write_chapter", ctx, deps)
        → write_chapter.run(ctx, deps)
            1. extract_write_chapter_args → chapter_id="1.1"
            2. _set_deps(deps)
            3. graph = build_writer_graph(checkpointer=...)
            4. graph.invoke(initial_state, config={"configurable": {"thread_id": session_id}})
               ┌─ prep_context: prep_context("1.1", task) → canon block
               ├─ plan_chapter: _call_plan_chapter → LLM 自由散文(WRITER_API_KEY 必填;deterministic raise)
               ├─ draft_chapter: prose_client.generate_text(system, user)
               ├─ proofread: 长度 / 格式 lint
               ├─ review_gate:
               │   - deterministic → ReviewVerdict(pass=True, score=8)
               │   - real → invoke_structured_json(ReviewVerdict) (>=7 通过)
               │   - 不通过或 rewrite=True 且 budget 内 → 回到 draft_chapter
               └─ persist_outputs:
                   - mkdir 草稿/ 目录
                   - write 草稿/chapter-1.1.md
                   - append_summary(project_root, "1.1", first_para, atomic=True)
            5. _reset_deps()
            6. _state_to_result(final_state) → WorkflowResult(
                 status="completed",
                 chunks=[trace, draft, proofread, review_gate],
                 artifacts={"draft_path": 草稿/chapter-1.1.md,
                            "summaries_path": chapter_summaries.json},
                 metrics={"score": 8, "retry_count": 0, ...})
   ↓
for chunk in result.chunks: yield TextChunk(chunk)
yield Done(reason="workflow_completed", payload={workflow, artifacts, metrics})
   ↓
CLI:
    console.print("[done] workflow_completed")
    session.record_turn("/创作 1.1 第一章", "workflow_completed")
```
