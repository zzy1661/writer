## 11.1 设计动机

**问题**:LLM 调用需要 `model` / `api_key` / `base_url` / `temperature` 等参数,这些参数:

- **用户级**(开发者本机全局):用户在 `/Users/me/.env` 写一份
- **项目级**(每个小说项目独立):`<project>/.writer/config` 写一份(项目用不同模型)
- **会话级**(临时):REPL 里临时覆盖

**配置优先级**(后者覆盖前者):

```
1. Shell 环境变量(WRITER_*)
2. 项目目录 .env
3. .writer/config(项目内最高优先级)
```

## 11.2 `Settings` — pydantic-settings 模型

> 对应代码:`src/writer/config/settings.py`

```python
from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_prefix="WRITER_",        # 只识别 WRITER_* 环境变量
        env_file=".env",
        env_file_encoding="utf-8",
        case_sensitive=False,
        extra="ignore",
    )

    # LLM Provider
    model: str = "gpt-4o-mini"
    api_key: str = ""
    base_url: str = "https://api.openai.com/v1"
    temperature: float = 0.7

    # 运行时
    log_level: str = "INFO"

    @property
    def has_api_key(self) -> bool:
        return bool(self.api_key)
```

### 关键字段

| 字段 | 类型 | 默认 | 含义 |
| ---- | ---- | ---- | ---- |
| `model` | `str` | `gpt-4o-mini` | LangChain 模型名 |
| `api_key` | `str` | `""` | API key(空 = rule-only 模式) |
| `base_url` | `str` | `https://api.openai.com/v1` | OpenAI 兼容端点 |
| `temperature` | `float` | `0.7` | LLM 温度 |
| `log_level` | `str` | `INFO` | Python `logging` 级别 |

### `has_api_key` 判定

```python
if settings.has_api_key:
    # 装配 CompositeRouter + ReActAgent + RealProseClient
else:
    # 仅装配 RuleBasedIntentRouter + tool_loop=None + DeterministicProseClient
```

`has_api_key` 是 engine / workflow / agent 的关键分支点。

## 11.3 配置加载顺序

```python
# src/writer/config/settings.py

_cached_settings: Settings | None = None


def get_settings() -> Settings:
    """全局单例:懒加载并缓存 settings。"""
    global _cached_settings
    if _cached_settings is None:
        _cached_settings = Settings()
    return _cached_settings


def refresh_settings() -> Settings:
    """强制重读 .env / .writer/config(项目切换时调)。"""
    global _cached_settings
    _cached_settings = None
    return get_settings()


def load_env_file(project_root: Path) -> None:
    """从 <project_root>/.writer/config 加载 env 风格配置。"""
    config_path = project_root / ".writer" / "config"
    if not config_path.exists():
        return
    for line in config_path.read_text(encoding="utf-8").splitlines():
        line = line.strip()
        if not line or line.startswith("#"):
            continue
        if "=" not in line:
            continue
        key, _, value = line.partition("=")
        key = key.strip()
        value = value.strip().strip('"').strip("'")
        os.environ[f"WRITER_{key}"] = value


def load_project_settings(project_root: Path) -> Settings:
    """CLI 启动时调:先 load_env_file(project_root),再 get_settings()。"""
    load_env_file(project_root)
    return get_settings()
```

### 加载流程

```
CLI 启动:
    project_root = discover_project_root()
    settings = load_project_settings(project_root)
        ├─ load_env_file(project_root)
        │     └─ 读 .writer/config → os.environ["WRITER_*"] = ...
        └─ Settings() → 读 os.environ → 实例化 Settings
   ↓
EngineSession 构造时:
    deps = production_deps(project_root=project_root)
        └─ resolved = settings if settings else get_settings()
   ↓
Engine 派发:
    if settings.has_api_key: 走 CompositeRouter + ReActAgent
    else: 走 RuleBasedIntentRouter + tool_loop=None
```

## 11.4 项目切换时的 refresh

```python
# EngineSession.set_project_root
def set_project_root(self, new_root: Path | None) -> None:
    if new_root == self.project_root:
        return
    if new_root is not None:
        from writer.config import load_env_file, refresh_settings
        load_env_file(new_root)
        refresh_settings()  # 强制重读,让新项目的 .writer/config 生效
    ...
```

**为什么必须 `refresh_settings()`**:`pydantic-settings` 的 `_cached_settings` 单例缓存了旧值;项目切换不刷新,新项目的配置永远不生效。

## 11.5 项目级 `.writer/config` 模板

`writer new` 在 `.writer/config` 写入模板:

```bash
# 项目级 LLM 配置(优先级高于 .env)
WRITER_MODEL=gpt-4o-mini
WRITER_API_KEY=
WRITER_BASE_URL=https://api.openai.com/v1
WRITER_TEMPERATURE=0.7
```

**用户编辑**:填入 `WRITER_API_KEY=sk-xxxxx`,REPL 启动自动读取。

## 11.8 完整数据流:用户在不同项目用不同模型

```
# 项目 A(用 GPT):
~/projects/novel-a/.writer/config:
    WRITER_MODEL=gpt-4o-mini
    WRITER_API_KEY=sk-gpt-...

cd ~/projects/novel-a
uv run writer
> /大纲 xxx
    load_env_file → WRITER_MODEL=gpt-4o-mini, WRITER_API_KEY=sk-gpt-...
    settings.model = "gpt-4o-mini"
    Engine 用 gpt-4o-mini 跑大纲

# 项目 B(用 DeepSeek):
~/projects/novel-b/.writer/config:
    WRITER_MODEL=deepseek-v3
    WRITER_API_KEY=sk-deepseek-...
    WRITER_BASE_URL=https://api.deepseek.com/v1

cd ~/projects/novel-b
uv run writer
> /大纲 xxx
    load_env_file → 重置 env vars
    refresh_settings → 重新读 Settings()
    settings.model = "deepseek-v3"
    needs_json_prompt_structured_output(settings) → True(模型名含 "deepseek")
    Engine 用 JSON-prompt 路径跑大纲
```
