## 3.1 设计动机

**为什么需要会话层?**Engine 是无状态的 `AsyncGenerator`,但 REPL 是有状态的:

- 多轮之间要共享 `session_id`(用于日志关联)
- `project_root` 会变化(`/init` 后才绑定)
- LLM 上轮提的问题(Interrupt)需要在下一轮拼接回 `user_input`
- 每个 turn 要记录到历史(用于 `/状态` 显示)

这些状态**不属于引擎**(它是纯函数式的输入→输出流),而属于「REPL 跨 turn 容器」。

**为什么不直接用 LangGraph checkpointer?**备忘 03 + 备忘 04 解释过:LangGraph 整套状态机还没接入生产代码(2026-07-09 的现状),Engine 用 AsyncGenerator 单层 dispatch 已经够用;LangGraph checkpointer 作为未来选项保留。本项目目前用 `EngineSession` + 追加式 `TurnRecord` 替代。

## 3.2 `EngineSession` 数据模型

```python
@dataclass
class EngineSession:
    # 冻结身份:构造时设一次
    session_id: UUID = field(default_factory=uuid4)
    started_at: datetime = field(default_factory=lambda: datetime.now(UTC))

    # 项目上下文(可变)
    project_root: Path | None = None
    project_state: str = "S0"        # 来自 detect_state()
    project_genre: str = "other"     # 从 AGENT.md 读

    # deps(构造时构建一次;project_root 变更时 rebind tool_runtime)
    deps: EngineDeps = field(default=None)

    # 仅追加的 turn 历史
    turns: list[TurnRecord] = field(default_factory=list)

    # 上轮的 Interrupt,消费后清空
    pending_interrupt: Interrupt | None = None
```

### 核心字段语义

| 字段 | 何时设置 | 何时变更 |
| ---- | -------- | -------- |
| `session_id` | 构造时(REPL 启动) | 永不 |
| `started_at` | 构造时 | 永不 |
| `project_root` | 构造时通过 `discover_project_root()` 自动探测;或 `/init` 后 CLI 回填 | `set_project_root(new)` 时 |
| `project_state` | 由 `detect_state(project_root)` 推导 | 任何写文件操作(隐式)→ `refresh_project_state()` |
| `project_genre` | 从 `AGENT.md` 的 `题材:` 行读 | `refresh_project_genre()` |
| `deps` | `__post_init__` 时 `production_deps(project_root=...)` | `set_project_root()` 时 rebind 4 个组件 |
| `turns` | 每次 `record_turn()` 追加 | 仅追加 |
| `pending_interrupt` | `set_pending_interrupt(interrupt)` | `clear_pending_interrupt()` |

## 3.3 项目状态机 S0–S5

`src/writer/project/state.py` 定义 6 个状态:

```python
class ProjectState(str, Enum):
    UNINITIALIZED = "S0"     # 无 AGENT.md
    INITIALIZED = "S1"       # 有 AGENT.md,但还没大纲
    OUTLINING = "S2"         # 有 outline/大纲.md
    FRAMING = "S3"           # 有 outline/toc.md(目录)
    DRAFTING = "S4"          # 有 manuscript/ 草稿
    REVIEWING = "S5"         # 当前卷审核中
```

### `detect_state()` 算法

```python
def detect_state(project_root: Path | None) -> ProjectState:
    if project_root is None or not (project_root / "AGENT.md").exists():
        return ProjectState.UNINITIALIZED
    # 倒序检测:状态 N 蕴含状态 N-1 都满足
    if _has_drafts(project_root):
        return ProjectState.DRAFTING
    if (project_root / "outline" / "toc.md").exists():
        return ProjectState.FRAMING
    if (project_root / "outline" / "大纲.md").exists():
        return ProjectState.OUTLINING
    return ProjectState.INITIALIZED
```

### 状态转移图

```
S0 ─/init <name>──> S1 ─/大纲──> S2 ─/目录──> S3 ─/创作──> S4 ─/审核+修复──> S5
                                                                 │
                                  (用户手动开启下一卷) <──────────┘
```

### 命令 × 状态矩阵(精简版)

| 命令 | S0 | S1 | S2 | S3 | S4 | S5 |
| ---- | -- | -- | -- | -- | -- | -- |
| `/init <name>` | ✓ | — | — | — | — | — |
| `/init <梗概>` | — | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/大纲` | — | ✓ | ✓ | △ | △ | △ |
| `/目录` | — | ✓ | ✓ | △ | △ | △ |
| `/创作` | — | — | — | ✓ | ✓ | ✓ |
| `/审核` | — | — | — | — | ✓ | ✓ |

详细矩阵见 [`docs/命令与用户流程.md §5.2`](../命令与用户流程.md)。

## 3.4 `validate_command_available` — 命令可用性校验

这是命令矩阵的**实现侧**入口,Engine 在派发前调用:

```python
def validate_command_available(
    command: str,
    project_root: Path | None,
    project_state: str,
    *,
    skill_registry: DirectiveRegistry | None = None,
) -> CommandAvailability:
    # 优先查 skill_registry.state_matrix()(SKILL.md requires_states 字段)
    if skill_registry is not None:
        matrix = skill_registry.state_matrix()
        if command in matrix:
            allowed = matrix[command]
            current = ProjectState(project_state)
            if current in allowed:
                return CommandAvailability(ok=True, state=current)
            return CommandAvailability(
                ok=False, state=current,
                reason=f"命令 {command} 在状态 {current.value} 不可用",
            )
    # fallback 到硬编码 COMMAND_ALLOWED
    return _legacy_check(command, project_root, project_state)
```

**关键点**:矩阵**自动派生**,SKILL.md frontmatter 写 `requires_states: [S1, S2]`,矩阵自动包含这条约束。

## 3.5 `set_project_root` — 项目热替换

这是 session 最复杂的方法。**当 `/init <name>` 跑完,payload 里有 `project_root`,CLI 就要调它**:

```python
def set_project_root(self, new_root: Path | None) -> None:
    if new_root == self.project_root:
        return  # no-op

    # 1. 加载新项目的 .env / config(若有)
    if new_root is not None:
        load_env_file(new_root)
        refresh_settings()

    self.project_root = new_root
    resolved = (new_root or _SENTINEL_PROJECT_ROOT).resolve()

    # 2. 重建 ToolRuntime(因为它持有 project_root)
    new_runtime = ToolRuntime(project_root=resolved)
    self.deps = self.deps.rebind_tool_runtime(new_runtime)

    # 3. 刷新项目元数据
    self.refresh_project_state()
    self.refresh_project_genre()

    # 4. 重建 DirectiveRegistry(让 .writer/skills/ 项目覆盖生效)
    new_registry = built_directive_registry(project_root=resolved)
    self.deps = self.deps.rebind_directive_registry(new_registry)

    # 5. 重建 AgentRegistry(让 .writer/agents/ 项目覆盖生效)
    new_agent_registry = built_agent_registry(project_root=resolved)
    self.deps = self.deps.rebind_agent_registry(new_agent_registry)

    # 6. 重建 LLMToolLoop(Bug 01 修复:必须用新 runtime 重建)
    from writer.llm.agent import LLMToolLoop
    new_loop: LLMToolLoop | None = None
    if self.deps.tool_loop is not None:
        new_loop = LLMToolLoop(
            settings=self.deps.settings,
            registry=self.deps.tool_registry,
            runtime=new_runtime,
        )
    self.deps = self.deps.rebind_tool_loop(new_loop)
```

### 关键设计

- **`rebind_*` 系列方法**:EngineDeps Protocol 上声明,`_DefaultEngineDeps` 用 `dataclasses.replace(self, ...)` 实现。**测试可以传非 dataclass 的实现**,因为 Protocol 不强制 dataclass。
- **保留 router / tool_registry**:这两个不依赖 project_root,不需要重建。
- **重建 LLMToolLoop**(Bug 01):LLMToolLoop 在构造时捕获 `self._runtime = runtime`,如果不在 project 切换时重建,会一直用旧 project_root 调 safe_path,导致新项目被错误拒绝。

## 3.7 Pending Interrupt 生命周期

引擎可以在 `ask_user` action 时产出 `Interrupt` 事件:

```python
yield Interrupt(type="text", prompt="请补充主角名字")
yield Done(reason="ask_user", payload={"prompt": "..."})
```

CLI 处理:

```python
case Interrupt():
    session.set_pending_interrupt(event)
case Done():
    if done.reason == "answered" and pending := session.pending_interrupt:
        # 用户回答了 pending question
        ...
        session.clear_pending_interrupt()
```

下一轮 `compose_pending_input` 会拼成:

```
[pending] 请补充主角名字
[answer] 张三
```

让 LLM 同时看到「上轮问了什么」+「用户回答」。

## 3.8 项目根自动探测 `discover_project_root()`

REPL 启动时调用,顺序:

```python
def discover_project_root() -> Path | None:
    cwd = Path.cwd()
    # 1. 当前目录含 AGENT.md
    if (cwd / "AGENT.md").exists():
        return cwd
    # 2. 当前目录下只有一个含 AGENT.md 的子目录
    candidates = [p for p in cwd.iterdir() if p.is_dir() and (p / "AGENT.md").exists()]
    if len(candidates) == 1:
        return candidates[0]
    # 3. 找不到 → None(用户需先 /init)
    return None
```

## 3.9 完整数据流:从 `/init <name>` 到下一轮 `/大纲`

```
REPL 启动:
    EngineSession(project_root=None, deps=production_deps())
    discover_project_root() → None

用户输入 "/init 长安程序员":
    handle_repl_input() → 交给 engine
    EngineContext(project_root=None, ...)
    deps.route() → AgentAction(action_type="run_command", command="/init")
    Engine 分派 → _run_init_command()
    create_workspace("长安程序员", Path("."))
    yield TextChunk("已初始化项目: ./长安程序员")
    yield Done(reason="answered", payload={"project_root": "/abs/path/长安程序员"})

CLI 收到 Done:
    session.set_project_root(Path("/abs/path/长安程序员"))
        ├─ rebuild ToolRuntime
        ├─ rebuild DirectiveRegistry(项目级 .writer/skills/ 已存在)
        ├─ rebuild AgentRegistry
        ├─ rebuild LLMToolLoop
        └─ refresh_project_state() → "S1"
        └─ refresh_project_genre() → "other"
    session.record_turn("/init 长安程序员", "answered")

下一轮 "/大纲 穿越唐朝":
    handle_repl_input() → 交给 engine
    EngineContext(project_root="/abs/path/长安程序员", project_state="S1", ...)
    deps.route() → AgentAction(command="/大纲")
    validate_command_available("/大纲", ..., state="S1") → ok
    directive_registry.get("/大纲") → 命中 _shipped/大纲/SKILL.md
    _run_directive() → 走 LLM 工具循环
    LLM 调 safe_read_file("/AGENT.md") → 读出题材
    LLM 调 safe_write_file("/outline/大纲.md", ...) → 写入大纲
    yield Done(reason="answered")
    CLI: session.record_turn(...) + session.refresh_project_state() → "S2"
```
