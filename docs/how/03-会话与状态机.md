## 3.1 设计动机

**为什么需要会话层?**Engine 是无状态的 `AsyncGenerator`,但 REPL 是有状态的:

- 多轮之间要共享 `session_id`(用于日志关联)
- `project_root` 会变化(`writer new <书名>` 或 `cd` 进入项目目录后绑定)
- LLM 上轮提的问题(Interrupt)需要在下一轮拼接回 `user_input`
- 每个 turn 要记录到历史(用于 `/状态` 显示)

这些状态**不属于引擎**(它是纯函数式的输入→输出流),而属于「REPL 跨 turn 容器」。

**为什么不直接用 LangGraph checkpointer?**备忘 03 + 备忘 04 解释过:LangGraph **已实装**为 `write_chapter`(PR2,2026-07-09)与 `review_chapter`(PR3,2026-07-09)两个 5 节点工作流内部状态机,Engine 自身仍用 AsyncGenerator 单层 dispatch 做顶层 turn 编排;LangGraph checkpointer 在工作流内部负责 checkpoint / resume,而 `Engine` 在 turn 层级负责会话状态。两者正交:LangGraph 维护单次工作流内部 state,`Engine` 维护跨 turn 会话状态。

## 3.2 `Engine` 数据模型

```python
@dataclass
class Engine:
    # 冻结身份:构造时设一次
    session_id: UUID = field(default_factory=uuid4)
    started_at: datetime = field(default_factory=lambda: datetime.now(UTC))

    # 项目上下文(可变)
    project_root: Path | None = None
    project_state: str = "S0"        # 来自 detect_state()
    project_genre: str = "other"     # 从 AGENT.md 读

    # engine:Engine 实例(构造时构建一次;project_root 变更时通过
    # Engine.replace_deps(new_deps) 整体替换内部 deps)
    engine: Engine | None = field(default=None)

    # 仅追加的 turn 历史
    turns: list[TurnRecord] = field(default_factory=list)

    # 上轮的 Interrupt,消费后清空
    pending_interrupt: Interrupt | None = None
```

### 核心字段语义

| 字段 | 何时设置 | 何时变更 |
| ---- | -------- | -------- |
| `session_id` | 构造时(REPL 启动) | 永不 |
| `started_at` | 构造时 | 永不 |
| `project_root` | 构造时通过 `discover_project_root()` 自动探测;或非 REPL 路径 `/init` 命令的 `answered` payload 回填(SDK / e2e pipe,REPL 不再通过 `/init` 创建项目) | `set_project_root(new)` 时 |
| `project_state` | 由 `detect_state(project_root)` 推导 | 任何写文件操作(隐式)→ `refresh_project_state()` |
| `project_genre` | 从 `AGENT.md` 的 `题材:` 行读 | `refresh_project_genre()` |
| `engine` | `__post_init__` 时构造 `Engine(deps=production_deps(...))` | `set_project_root()` 通过 `Engine.replace_deps(new_deps)` 整体替换 deps |
| `turns` | 每次 `record_turn()` 追加 | 仅追加 |
| `pending_interrupt` | `set_pending_interrupt(interrupt)` | `clear_pending_interrupt()` |

## 3.3 项目状态机 S0–S5

`src/writer/project/state.py` 定义 6 个状态:

```python
class ProjectState(str, Enum):
    UNINITIALIZED = "S0"     # 无 AGENT.md
    INITIALIZED = "S1"       # 有 AGENT.md,但还没大纲
    OUTLINING = "S2"         # 有 outline/大纲.md
    FRAMING = "S3"           # 有 outline/toc.md(目录)
    DRAFTING = "S4"          # 有 manuscript/ 草稿
    REVIEWING = "S5"         # 当前卷审核中
```

### `detect_state()` 算法

```python
def detect_state(project_root: Path | None) -> ProjectState:
    if project_root is None or not (project_root / "AGENT.md").exists():
        return ProjectState.UNINITIALIZED
    # 倒序检测:状态 N 蕴含状态 N-1 都满足
    if _has_drafts(project_root):
        return ProjectState.DRAFTING
    if (project_root / "outline" / "toc.md").exists():
        return ProjectState.FRAMING
    if (project_root / "outline" / "大纲.md").exists():
        return ProjectState.OUTLINING
    return ProjectState.INITIALIZED
```

### 状态转移图

```
S0 ─writer new──> S1 ─/大纲──> S2 ─/目录──> S3 ─/创作──> S4 ─/审核+修复──> S5
                                                                  │
                                  (用户手动开启下一卷) <────────────┘

S0 → S1 由 Typer 子命令 `writer new <书名>` 完成,REPL 不再提供
`/init <name>` flag 形式(per 2026-07-14)。`/init <故事核心创意>` 是
S1+ 路径(对已绑定项目写 brief),不进状态机。
```

### REPL `/init` × 状态可用性(精简版,per 2026-07-14 收紧)

| 命令 | S0 | S1 | S2 | S3 | S4 | S5 |
| ---- | -- | -- | -- | -- | -- | -- |
| `writer new <name>`(Typer 子命令) | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/init <故事梗概>`(REPL) | — | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/大纲` | — | ✓ | ✓ | △ | △ | △ |
| `/目录` | — | ✓ | ✓ | △ | △ | △ |
| `/创作` | — | — | — | ✓ | ✓ | ✓ |
| `/审核` | — | — | — | — | ✓ | ✓ |

> `/init` 形式已收紧:命令后只跟故事核心创意;创建项目的唯一入口是
> Typer 子命令 `writer new <书名>`。S0 用户输入 `/init <故事梗概>`
> 时,REPL 走引导:`"请先执行 writer new <书名> 创建项目,再 cd 进入项目
> 目录后使用 /init <故事梗概>"`,不创建项目目录。

详细矩阵见 [`docs/命令与用户流程.md §5.2`](../命令与用户流程.md)。

## 3.4 `validate_command_available` — 命令可用性校验

这是命令矩阵的**实现侧**入口,Engine 在派发前调用:

```python
def validate_command_available(
    command: str,
    project_root: Path | None,
    project_state: str,
    *,
    skill_registry: DirectiveRegistry | None = None,
) -> CommandAvailability:
    # 优先查 skill_registry.state_matrix()(SKILL.md requires_states 字段)
    if skill_registry is not None:
        matrix = skill_registry.state_matrix()
        if command in matrix:
            allowed = matrix[command]
            current = ProjectState(project_state)
            if current in allowed:
                return CommandAvailability(ok=True, state=current)
            return CommandAvailability(
                ok=False, state=current,
                reason=f"命令 {command} 在状态 {current.value} 不可用",
            )
    # fallback 到硬编码 COMMAND_ALLOWED
    return _legacy_check(command, project_root, project_state)
```

**关键点**:矩阵**自动派生**,SKILL.md frontmatter 写 `requires_states: [S1, S2]`,矩阵自动包含这条约束。

## 3.5 `set_project_root` — 项目热替换

这是 session 最复杂的方法。**当非 REPL 路径 `/init <name>` 走完(或 REPL 启动时 `discover_project_root()` 探测成功),CLI 要调它**让 session 绑定项目根。

> per 2026-07-14:REPL `/init --name X --dir Y` flag 形式已删除,REPL 内不再产生需要 `set_project_root` 回填的 `answered` payload。REPL 项目绑定仅在 REPL 启动时 `discover_project_root()` 自动探测。Typer 子命令 `writer new <书名>` 仍可被脚本调用,创建项目后用户需 `cd` 进项目目录,REPL 启动时自动绑定。

```python
def set_project_root(self, new_root: Path | None) -> None:
    if new_root == self.project_root:
        return  # no-op

    # 1. 加载新项目的 .env / config(若有)
    if new_root is not None:
        load_env_file(new_root)
        refresh_settings()

    self.project_root = new_root
    resolved = (new_root or _SENTINEL_PROJECT_ROOT).resolve()

    # 2. 重建 ToolRuntime(因为它持有 project_root)
    new_runtime = ToolRuntime(project_root=resolved)
    new_deps = self.engine.deps.rebind_tool_runtime(new_runtime)

    # 3. 刷新项目元数据
    self.refresh_project_state()
    self.refresh_project_genre()

    # 4. 重建 DirectiveRegistry(让 .writer/skills/ 项目覆盖生效)
    new_registry = built_directive_registry(project_root=resolved)
    new_deps = new_deps.rebind_directive_registry(new_registry)

    # 5. 重建 AgentRegistry(让 .writer/agents/ 项目覆盖生效)
    new_agent_registry = built_agent_registry(project_root=resolved)
    new_deps = new_deps.rebind_agent_registry(new_agent_registry)

    # 6. 重建 ReActAgent(Bug 01 修复:必须用新 runtime 重建)
    from writer.llm.agent import ReActAgent
    new_loop: ReActAgent | None = None
    if new_deps.tool_loop is not None:
        new_loop = ReActAgent(
            settings=new_deps.settings,
            registry=new_deps.tool_registry,
            runtime=new_runtime,
        )
    new_deps = new_deps.rebind_tool_loop(new_loop)

    # 7. 整体替换 engine 内部的 deps(cfg / router / tool_registry 保持不变)
    self.engine = self.engine.replace_deps(new_deps)
```

### 关键设计

- **`rebind_*` 系列方法**:RunnerDeps Protocol 上声明,`_DefaultRunnerDeps` 用 `dataclasses.replace(self, ...)` 实现。**测试可以传非 dataclass 的实现**,因为 Protocol 不强制 dataclass。
- **`Engine.replace_deps(new_deps)`**:整体替换 `Engine` 实例的内部 deps(per 2026-07-13 `Engine` 类重构);`cfg` 保持不变(同进程内 `fast_mode` 等配置不变)。`router` / `tool_registry` 不依赖 project_root,所以不需要 rebuild。
- **重建 ReActAgent**(Bug 01):ReActAgent 在构造时捕获 `self._runtime = runtime`,如果不在 project 切换时重建,会一直用旧 project_root 调 safe_path,导致新项目被错误拒绝。

## 3.7 Pending Interrupt 生命周期

引擎可以在 `ask_user` action 时产出 `Interrupt` 事件:

```python
yield Interrupt(type="text", prompt="请补充主角名字")
yield Done(reason="ask_user", payload={"prompt": "..."})
```

CLI 处理:

```python
case Interrupt():
    session.set_pending_interrupt(event)
case Done():
    if done.reason == "answered" and pending := session.pending_interrupt:
        # 用户回答了 pending question
        ...
        session.clear_pending_interrupt()
```

下一轮 `compose_pending_input` 会拼成:

```
[pending] 请补充主角名字
[answer] 张三
```

让 LLM 同时看到「上轮问了什么」+「用户回答」。

## 3.8 项目根自动探测 `discover_project_root()`

REPL 启动时调用,顺序:

```python
def discover_project_root() -> Path | None:
    cwd = Path.cwd()
    # 1. 当前目录含 AGENT.md
    if (cwd / "AGENT.md").exists():
        return cwd
    # 2. 当前目录下只有一个含 AGENT.md 的子目录
    candidates = [p for p in cwd.iterdir() if p.is_dir() and (p / "AGENT.md").exists()]
    if len(candidates) == 1:
        return candidates[0]
    # 3. 找不到 → None(用户需先 /init)
    return None
```

## 3.9 完整数据流:从 `writer new <书名>` 到下一轮 `/大纲`

> per 2026-07-14:`writer new <书名>` 是 Typer 子命令,不走 REPL。

```
Shell:
    uv run writer new 长安程序员 -g 历史
        → cli/commands.py::new_project_cmd
        → create_new_workspace("长安程序员", Path("."), genres=["历史"])
        → 创建项目目录 + AGENT.md + 题材行,console.print 列出 created_files
        → 退出。

REPL 启动 (cd 进项目目录后 uv run writer):
    Engine(engine=Engine(deps=production_deps()))
    discover_project_root() → "/abs/path/长安程序员"(自动绑定)
    session.set_project_root("/abs/path/长安程序员")
        ├─ rebuild ToolRuntime
        ├─ rebuild DirectiveRegistry(项目级 .writer/skills/ 已存在)
        ├─ rebuild AgentRegistry
        ├─ rebuild ReActAgent
        ├─ refresh_project_state() → "S1"
        └─ refresh_project_genre() → "历史"
        └─ Engine.replace_deps(new_deps) → 整体替换 engine 内部 deps(cfg 不变)
    REPL 控制台打印:已自动绑定项目: ...(初始化)

用户输入 "/init 一个程序员穿越到唐朝,用代码改造官僚体系":
    handle_repl_input() → _try_handle_repl_init_brief(text, session)
        ├─ project_root = session.project_root or discover_project_root() → "长安程序员"
        ├─ prompt_genres(console, default=None) → TTY 交互 / 非 TTY ["其他"]兜底
        ├─ apply_genre_and_brief(project_root, genres=selected, brief=rest, settings=...)
        │       ├─ apply_genre_scaffolding(root, genres) → 补脚手架(已存在 -> no-op)
        │       ├─ update_agent_genre_line(AGENT.md, genres) → 更新题材行
        │       └─ apply_init_brief → 写 创意/核心创意.md + 追加 ## 基本要求 段
        └─ session.refresh_project_genre()
    console.print 摘要:所选题材 / 新建文件数 / 题材行变更 / brief 来源
    return True (已消费,不再交给 engine)

下一轮 "/大纲 穿越唐朝":
    handle_repl_input() → 交给 engine
    session.run_turn(user_input) 构造 RunnerContext(project_root="/abs/path/长安程序员", project_state="S1", ...)
    async for event in session.engine.run(ctx):
        Engine._engine_loop:
            self._deps.route() → AgentAction(command="/大纲")
            directive = self._deps.directive_registry.get("/大纲") → 命中 _shipped/大纲/SKILL.md
            self._run_directive() → 走 LLM 工具循环
            LLM 调 safe_read_file("/AGENT.md") → 读出题材 "历史"
            LLM 调 safe_write_file("/大纲/大纲.md", ...) → 写入大纲
            yield Done(reason="answered")
    CLI: session.record_turn(...) + session.refresh_project_state() → "S2"
```
