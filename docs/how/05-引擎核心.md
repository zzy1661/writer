## 5.1 设计动机

**问题**:一次 REPL turn 包含多个阶段:路由 → 派发 → 工具调用 → LLM 循环 → 终止。中间要产出**多种事件**(文本、动作、工具调用、工具结果、错误、终止)。

**三种做法对比**:

| 方案 | 缺点 |
| ---- | ---- |
| 单函数返回 `Result` | 不能流式输出;错误处理不灵活 |
| 回调注册一堆 listener | 类型不安全;复杂 |
| **AsyncGenerator 事件流**(本项目) | 流式 + 取消友好 + 简单 match |

`Engine.run(ctx)` 是 `AsyncIterator[Event]`,REPL 用 `async for event in engine.run(ctx)` 逐事件消费。引擎主类为 `Engine`(`src/writer/engine/engine.py`),持有 `RunnerDeps` + `RunnerConfig`;`engine.loop.run_runner(ctx, deps)` 是 compat shim(构造临时 `Engine(deps, cfg)` 委派给 `engine.run(ctx)`)。新代码应直接持有 `Engine` 实例(典型来源 `Engine.engine`)。

## 5.2 事件层级

> 对应代码:`src/writer/engine/events.py`

```python
DoneReason = Literal[
    "answered",              # LLM 给文本答案
    "command_pending",       # 静态命令尚未实装
    "tool_pending",          # 保留分支(目前未使用)
    "ask_user",              # 需要 Interrupt
    "aborted",               # 错误兜底
    "tool_completed",        # 同步 _run_tool 完成
    "tool_loop_completed",   # ReActAgent 预算耗尽
    "workflow_completed",    # workflow 跑完
]


@dataclass(frozen=True)
class Event: ...


@dataclass(frozen=True)
class TextChunk(Event):
    """一段人类可读文本,可能后跟更多文本块。"""
    text: str


@dataclass(frozen=True)
class ActionEvent(Event):
    """派发器已为当前输入产出 AgentAction。"""
    action: AgentAction


@dataclass(frozen=True)
class ToolCall(Event):
    name: str
    arguments: dict[str, Any]


@dataclass(frozen=True)
class ToolResult(Event):
    name: str
    output: str


@dataclass(frozen=True)
class Interrupt(Event):
    type: Literal["choice", "text", "confirm"]
    prompt: str
    options: list[str] | None = None


@dataclass(frozen=True)
class Done(Event):
    reason: DoneReason
    payload: dict[str, Any] | None = None


@dataclass(frozen=True)
class ErrorEvent(Event):
    message: str
    traceback: str | None = None  # 2026-07-05 arch-optimizer M4 增补
```

### 关键约束

- **全部 `@dataclass(frozen=True)`**:消费者可放心 `match` 而无需防御性拷贝
- **`AgentAction` 是 Pydantic 不是 dataclass**:第 4 章讲过,新人不要用 `dataclasses.replace`
- **`payload: dict | None`**:Done 的 payload 携带 turn 摘要(`init_brief=True` / `tool_calls_made=3` / `error="..."` 等),CLI 据此渲染

### `production_deps()` — 纯工厂

```python
def production_deps(

    *,
    project_root: Path | None = None,

    resolved = settings if settings is not None else get_settings()

        prose_client = DeterministicProseClient()

    resolved_agent_registry = agent_registry or built_agent_registry(project_root=root)

    return _DefaultRunnerDeps(
        router=_select_router(resolved, primary=primary_router, agent_registry=resolved_agent_registry),
        agent_registry=resolved_agent_registry,
        tool_registry=tool_registry,

        directive_registry=built_directive_registry(project_root=root),
```

**关键设计**:
- 纯工厂:**不在调用方背后做文件系统 IO**(`project_root` 显式传)
- 延迟 import:`from writer.llm.agent import ReActAgent` 放在 `if resolved.has_api_key` 块内,纯规则部署不加载 LLM 客户端栈
- `_NO_PROJECT_ROOT` 哨兵:S0 路径下用 `Path("/__no_project__")`,让 `safe_path` 仍能拒绝越界

## 5.5 `_engine_loop` — `Engine` 类的核心派发私有方法

> 对应代码:`src/writer/engine/engine.py::Engine._engine_loop`

整体 body 被 `try/except` 包裹,三层 except(`ToolError` / `SkillError` / `Exception`)产出 `ErrorEvent + Done(aborted)`。所有派发 helper(`_run_tool` / `_run_tool_loop` / `_run_workflow` / `_run_agent` / `_run_directive` / `_maybe_run_init_brief_or_block` / `_run_init_brief_command` / `_run_init_command`)都是 `Engine` 类的私有方法,从 `self._deps` / `self._cfg` 读依赖,无需每个 helper 都传 `(action, ctx, deps, cfg)`。

### 伪代码(简化)

```python
class Engine:
    async def _engine_loop(self, ctx):
        deps = self._deps
        cfg = self._cfg
        try:
            action = deps.route(ctx.user_input, ctx.project_state)
            yield ActionEvent(action=action)

            # /init 特殊路径: REPL 抢先消费(2026-07-13)已经处理过 /init <brief>
            # 形式;这里只服务 S0 引导 + 非 REPL 调用方(e2e / SDK / 脚本)。
            # per 2026-07-14: REPL /init --flag 形式已删除,创建项目请用
            # Typer 子命令 `writer new <书名>`。
            if action.command == "/init" and action.action_type == "run_command":
                async for event in self._maybe_run_init_brief_or_block(ctx):
                    yield event
                if _init_turn_handled(ctx.user_input, ctx.project_root, ctx.project_state):
                    return

            # kind="agent" 优先派发(per fea-agent-mirror)
            if action.kind == "agent":
                async for event in self._run_agent(action, ctx):
                    yield event
                return

            # 按 action.action_type 分派
            match action.action_type:
                case "answer_directly":
                    yield TextChunk(text=action.answer or "")
                    yield Done(reason="answered", payload={"answer": action.answer})

                case "run_command":
                    if action.command == "/init":
                        async for event in self._run_init_command(ctx):
                            yield event
                    elif directive := deps.directive_registry.get(action.command):
                        # 动态派发:任何已注册 directive 都走 LLM 工具循环
                        async for event in self._run_directive(directive, ctx):
                            yield event
                    else:
                        yield Done(reason="command_pending", payload={"command": action.command})

                case "call_tool":
                    if deps.tool_loop is not None:
                        async for event in self._run_tool_loop(action, ctx):
                            yield event
                    else:
                        async for event in self._run_tool(action):
                            yield event

                case "start_workflow":
                    async for event in self._run_workflow(action.workflow or "", ctx):
                        yield event

                case "ask_user":
                    yield Interrupt(type="text", prompt=action.user_prompt or "请补充信息")
                    yield Done(reason="ask_user", payload={"prompt": action.user_prompt})

        except ToolError as exc:
            yield ErrorEvent(message=f"工具错误: {exc}", traceback=traceback.format_exc())
            yield Done(reason="aborted", payload={"error": str(exc)})
        except SkillError as exc:
            yield ErrorEvent(message=f"技能错误: {exc}", traceback=traceback.format_exc())
            yield Done(reason="aborted", payload={"error": str(exc), "command": getattr(exc, "command", None)})
        except Exception as exc:
            yield ErrorEvent(message=f"引擎异常: {exc}", traceback=traceback.format_exc())
            yield Done(reason="aborted", payload={"error": str(exc)})
```

### 关键设计

- **`match` 语句单层 dispatch**:Python 3.10+ 的 `match/case` 让分派清晰
- **三层 except 兜底**:`ToolError` → `SkillError` → `Exception`,从领域异常到兜底异常
- **traceback 始终进 payload**:用户可以直接贴进 bug report,无需重跑 + debugger

## 5.6 `Done` 八种 reason 详解

| reason | 触发条件 | payload 关键字段 | CLI 渲染 |
| ------ | -------- | ---------------- | -------- |
| `answered` | `answer_directly` / `/init` 成功 / SKILL.md 跑完 | `answer` / `project_root` | 打印文本 + 调 `session.record_turn` |
| `command_pending` | 静态命令尚未实装 | `command` | 提示「待实现」 |
| `tool_pending` | 保留分支(目前未使用) | — | — |
| `ask_user` | LLM 问用户 | `prompt` | 提示用户输入 |
| `aborted` | 异常兜底 | `error` / `command` | 红色错误信息 + traceback |
| `tool_completed` | 同步 `_run_tool` 走完 | `tool` / `output` | 工具调用日志 |
| `tool_loop_completed` | `ReActAgent` 预算耗尽(5 步) | `tool_calls_made` / `last_output` | 提示「预算耗尽,请继续」 |
| `workflow_completed` | workflow `status="completed"` | `workflow` / `artifacts` / `metrics` | 工作流产物列表 |

### `Engine._run_tool` — 同步 Tool 调用

```python
class Engine:
    async def _run_tool(self, action):
        deps = self._deps
        cfg = self._cfg
        name = action.tool_name or ""
        arguments = dict(action.arguments)
        yield ToolCall(name=name, arguments=arguments)
        result = deps.tool_registry.invoke(name, deps.tool_runtime, **arguments)
        yield ToolResult(name=name, output=result.output)
        yield Done(reason="tool_completed", payload={"tool": name, "output": result.output})
```

```python
class Engine:

        if deps.tool_loop is None:
```

```python
class Engine:
    async def _run_directive(self, directive, ctx):
        deps = self._deps
        cfg = self._cfg
        from writer.skills.directive_discovery import resolve_references
        resolved = resolve_references(directive.body, directive.references)

        if deps.tool_loop is not None:
            # 把 directive body 喂给 LLM,让它用 tool registry 完成工作

                command=directive.command,
                answer=directive.body,

                yield event
        else:

            yield Done(reason="answered", payload={"directive": directive.command, ...})
```

### `Engine._run_agent` — agent 形态派发

```python
class Engine:
    async def _run_agent(self, action, ctx):
        from writer.agents import AgentRegistryError
        deps = self._deps
        cfg = self._cfg
        agent_name = action.target_agent or ""
        try:
            agent = deps.agent_registry.require(agent_name)
        except AgentRegistryError as exc:
            yield ErrorEvent(message=f"Agent 错误: {exc}", traceback=str(exc))
            yield Done(reason="aborted", payload={"error": str(exc), "command": agent_name})
            return

        if deps.tool_loop is not None:
            # 把 agent body 作为 system identity,user input 作为 human message
            agent_action = AgentAction(
                action_type="answer_directly",
                command=None,
                kind="agent",
                target_agent=agent_name,
                answer=f"[agent {agent_name!r} system identity]\n{agent.body}\n\n[user input]\n{ctx.user_input}",
            )
            async for event in deps.tool_loop.run(agent_action, ctx, deps, cfg):
                yield event
        else:
            # 纯规则部署:输出 agent 预览
            yield TextChunk(text=f"[agent {agent_name!r} preview, no LLM configured]\n...")
            yield Done(reason="answered", payload={"agent": agent_name, "llm_available": False})
```

### `Engine._run_workflow` — workflow 派发

```python
class Engine:
    async def _run_workflow(self, name, ctx):
        deps = self._deps
        cfg = self._cfg
        yield TextChunk(text=f"[engine] 工作流 {name} 启动\n")
        result = deps.run_workflow(name, ctx)
        for chunk in result.chunks:
            yield TextChunk(text=chunk)
        if result.status == "completed":
            yield Done(reason="workflow_completed", payload={"workflow": name, "artifacts": {...}, "metrics": ...})
        elif result.status == "pending":
            # needs_rewrite 信号 → 以 aborted 暴露,带 decision metric
            decision = str(result.metrics.get("decision", "needs_rewrite"))
            yield Done(reason="aborted", payload={"workflow": name, "decision": decision})
        else:  # "failed"
            yield Done(reason="aborted", payload={"workflow": name, "error": ...})
```

## 5.9 引擎边界错误处理细则

```python
except ToolError as exc:
    """Tool 业务异常:路径越界、权限、IO 错误、未找到工具等。
    Tool 内部把 IO 错误转 ToolResult,通常不外溢;能到达这里的通常是
    schema 错误 / permission / not found。"""
    tb = traceback.format_exc()
    log.warning("工具错误: %s", exc, exc_info=True)
    yield ErrorEvent(message=f"工具错误: {exc}", traceback=tb)
    yield Done(reason="aborted", payload={"error": str(exc)})

except SkillError as exc:
    """Skill 侧异常:缺失 project root、前置条件未满足、参数格式错误。
    SkillError 自动携带被拒绝的命令,从最近派发的 action 恢复。"""
    tb = traceback.format_exc()
    log.warning("技能错误: %s", exc, exc_info=True)
    command = getattr(exc, "command", getattr(action, "command", None))
    yield ErrorEvent(message=f"技能错误: {exc}", traceback=tb)
    yield Done(reason="aborted", payload={"error": str(exc), "command": command})

except Exception as exc:
    """任何意外:绝不让异常从 async generator 冒泡出去。"""
    tb = traceback.format_exc()
    log.exception("引擎边界异常: %s", exc)
    yield ErrorEvent(message=f"引擎异常: {exc}", traceback=tb)
    yield Done(reason="aborted", payload={"error": str(exc)})
```

**关键约定**:**`ErrorEvent.message` 是人读摘要,`ErrorEvent.traceback` 是格式化堆栈**。CLI 渲染时 message + traceback 一起输出,用户可直接复制 traceback 贴进 bug report。

## 5.10 完整数据流:`/大纲 穿越唐朝`

```
用户输入 "/大纲 穿越到唐朝"
   ↓
Engine.run_turn(user_input) → 构造 RunnerContext 并委派给 Engine.run(ctx)
   ↓
RunnerContext(user_input="/大纲 穿越到唐朝", project_root=..., project_state="S1", session_id=...)
   ↓
async for event in session.engine.run(ctx):   # Engine.run(ctx)
    ↓ (Engine 内部 _engine_loop)
self._deps.route(user_input, project_state) → AgentAction(action_type="run_command", command="/大纲")
   ↓
yield ActionEvent(...)
   ↓
directive = self._deps.directive_registry.get("/大纲")  # 命中 _shipped/大纲/SKILL.md
   ↓
async for event in self._run_directive(directive, ctx):
    resolved = resolve_references(directive.body, directive.references)
    # 构造 agent_action 让 LLM 读 directive body
    agent_action = AgentAction(action_type="answer_directly", command="/大纲", answer=directive.body)
    async for event in self._deps.tool_loop.run(agent_action, ctx, self._deps, self._cfg):
        # ReActAgent 进入 ReAct 循环
        yield TextChunk(...) / ToolCall(...) / ToolResult(...) / Done("answered")
```
