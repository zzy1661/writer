## 4.1 设计动机

**问题**:用户输入是一行字符串(`/大纲 xxx` / `帮我写个大纲` / `退出` / 自由文本)。引擎需要一个**确定性的、结构化的决策**:`AgentAction`。

**传统做法**:把字符串丢给 LLM 让它分类。

**本项目做法**:**双层路由(rule-first + LLM fallback)**。理由:

| 维度 | rule-only | LLM-only | rule-first + LLM fallback(我们的选择) |
| ---- | --------- | -------- | ------------------------------------ |
| 离线可用 | ✓ | ✗ | ✓ |
| 延迟 | 0 | 1-3s | 0(命中规则)/ 1-3s(fallback) |
| 鲁棒性 | 完全确定 | 受 prompt 影响 | 高:规则路径确定性高,LLM 路径兜底 |
| 扩展性 | 改代码 | 改 prompt | 新命令加规则,新意图走 LLM |

## 4.2 `AgentAction` — 路由输出

```python
class AgentAction(BaseModel):
    model_config = {"frozen": True}

    action_type: Literal[
        "run_command",     # 派发给 directive_registry / init / 静态命令
        "call_tool",       # 直接调 Tool
        "start_workflow",  # 启动 workflow
        "ask_user",        # 需要 Interrupt
        "answer_directly", # 直接给文本答案
    ]
    command: str | None = None        # /init / 大纲 / 目录 / ...
    role: Role | None = None          # story_agent / proofreader / historian / reviewer
    workflow: str | None = None       # write_chapter / review_chapter
    tool_name: str | None = None      # safe_read_file / wordcount / ...
    arguments: dict[str, Any] = {}
    answer: str | None = None         # answer_directly 的文本
    user_prompt: str | None = None    # ask_user 的提示
    kind: Literal["command", "agent"] = "command"  # fea-agent-mirror 增补
    target_agent: str | None = None   # kind="agent" 时填充
```

### 为什么用 Pydantic BaseModel 而不是 dataclass?

1. **JSON 序列化廉价**:未来 `LlmIntentRouter` 用 LangChain structured output 产 `AgentAction` 时,JSON 解析直接走 Pydantic schema 校验
2. **frozen=True**:实例不可变,引擎 / LLM 拿到后无需防御性拷贝
3. **`model_copy(update=...)`**:代替 `dataclasses.replace(...)`,**新人不要用 replace**

### `kind` 字段(2026-07-09 fea-agent-mirror 增补)

新增的 `kind` 字段区分两种 action:
- `kind="command"`(默认):传统命令形态,填充 `command` / `workflow` / `tool_name`
- `kind="agent"`:agent 形态,填充 `target_agent`,LLM 按 agent identity 回答

引擎分支(由 `Engine._engine_loop` 实现):

```python
# Engine 类内部
if action.kind == "agent":
    async for event in self._run_agent(action, ctx):
        yield event
    return  # 不再走 match action.action_type

match action.action_type:
    case "answer_directly": ...
    case "run_command": ...
    ...
```

## 4.3 `IntentRouter` Protocol

```python
@runtime_checkable
class IntentRouter(Protocol):
    def route(self, user_input: str, project_state: str) -> AgentAction: ...
```

**故意忽略 `project_state`**:rule 版不读它,保留这个参数是为了让 `LlmIntentRouter` 接入时 Protocol 不变。

**关键设计**:`@runtime_checkable` 让 `isinstance(obj, IntentRouter)` 能工作,但**只看 attribute presence,不查类型**。测试时 `obj` 必须有 `route` 方法(签名可以更宽松)。

## 4.4 `RuleBasedIntentRouter` — 无网络 MVP

无依赖、纯规则、确定性。

### 核心代码

```python
class RuleBasedIntentRouter:
    _FRAMEWORK_KEYWORDS: frozenset[str] = frozenset({"init", "状态", "退出", "帮助"})

    def route(self, user_input: str, project_state: str) -> AgentAction:
        del project_state
        text = user_input.strip()

        # 1. 静态命令:走 run_command
        if text.startswith("/字数统计"):
            return AgentAction(
                action_type="call_tool",
                command="/字数统计",
                role="story_agent",
                tool_name="wordcount",
                arguments={"path": _command_argument(text, "/字数统计") or "."},
            )
        # 2. 工作流命令:走 start_workflow
        if text.startswith("/创作"):
            return AgentAction(
                action_type="start_workflow",
                command="/创作",
                role="story_agent",
                workflow="write_chapter",
                arguments={"raw": text},
            )
        if text.startswith("/审核"):
            return AgentAction(
                action_type="start_workflow",
                command="/审核",
                role="reviewer",
                workflow="review_chapter",
                arguments={"raw": text},
            )
        # 3. 伏笔查询:正则提取 F\d+
        if "伏笔" in text or "F0" in text:
            return AgentAction(
                action_type="call_tool",
                role="story_agent",
                tool_name="foreshadow_search",
                arguments=_parse_foreshadow_args(text),
            )
        # 4. 其他斜杠命令:尝试 directive registry 命中
        if text.startswith("/"):
            return AgentAction(
                action_type="run_command",
                command=text.split(maxsplit=1)[0],
            )
        # 5. 自然语言:走 LLM fallback(CompositeRouter 在这里)
        return AgentAction(
            action_type="answer_directly",
            answer="我可以处理 /init、/大纲、/目录、/创作、/审核 等写作命令。..."
        )
```

### `looks_like_command`

`CompositeRouter` 用它判断要不要走 LLM fallback:

```python
@classmethod
def looks_like_command(cls, text: str) -> bool:
    stripped = text.strip()
    if not stripped:
        return False
    if stripped.startswith("/"):
        return True
    return stripped in cls._FRAMEWORK_KEYWORDS
```

**故意保守**:误报(返回 True 但其实不是命令)只多花一次 LLM 调用;漏报会破坏规则优先契约。

## 4.5 `_parse_foreshadow_args` — 伏笔查询参数抽取

```python
_FID_PATTERN = re.compile(r"\bF\d+\b")

def _parse_foreshadow_args(text: str) -> dict[str, Any]:
    stripped = text.strip()
    match = _FID_PATTERN.search(stripped)
    if match is None:
        return {"keyword": stripped}
    return {"id": match.group(0), "keyword": stripped}
```

**关键**:从 "查一下 F003 那条玉佩的伏笔" 抽出 `id="F003"` + `keyword="查一下 F003 那条玉佩的伏笔"`,让 `foreshadow_search` Tool 同时按 id 和 keyword 双发查询。

## 4.6 `LlmIntentRouter` — LangChain 结构化输出

> 对应代码:`src/writer/routing/llm_router.py`

LLM 看到用户输入 + 系统 prompt(含可用 agent 列表),产出结构化 `AgentAction`。

### 双 provider 路径

```python
class LlmIntentRouter:
    def __init__(self, settings: Settings, agent_registry: AgentRegistry):
        self._use_json_prompt = needs_json_prompt_structured_output(settings)
        self._llm = get_llm(settings)
        if self._use_json_prompt:
            self._system_prompt = self._build_json_prompt(agent_registry)
        else:
            self._llm = self._llm.with_structured_output(AgentAction)
            self._system_prompt = _build_native_prompt(agent_registry)

    def route(self, user_input: str, project_state: str) -> AgentAction:
        messages = [
            SystemMessage(content=self._system_prompt),
            HumanMessage(content=user_input),
        ]
        if self._use_json_prompt:
            # JSON-prompt path:模型输出 JSON,后端解析
            response = self._llm.invoke(messages)
            return AgentAction.model_validate_json(response.content)
        # Native path:OpenAI 兼容协议原生结构化输出
        return self._llm.invoke(messages)
```

### 切换条件:`needs_json_prompt_structured_output(settings)`

> 对应代码:`src/writer/llm/structured.py`

```python
def needs_json_prompt_structured_output(settings: Settings) -> bool:
    """某些 provider 拒绝 native tool binding,需要 fallback 到 JSON-prompt。"""
    return "deepseek" in settings.model.lower()  # 启发式,未来可配
```

**启发式判断**:`deepseek-v3` 等已知不支持原生 tool binding 的模型 → 走 JSON-prompt;OpenAI / GPT 等 → 走原生。

### Native path:JSON-prompt 输出形态

```json
{
  "action_type": "call_tool",
  "tool_name": "safe_read_file",
  "arguments": {"path": "AGENT.md"}
}
```

后端用 Pydantic `AgentAction.model_validate_json(json_str)` 校验并实例化。

## 4.7 `CompositeRouter` — rule-first + LLM fallback

> 对应代码:`src/writer/routing/composite_router.py`

```python
class CompositeRouter:
    def __init__(self, primary: IntentRouter, fallback: IntentRouter):
        self._primary = primary
        self._fallback = fallback

    def route(self, user_input: str, project_state: str) -> AgentAction:
        # 1. 主路由:rule-based
        action = self._primary.route(user_input, project_state)
        # 2. 如果主路由把自然语言归类为 answer_directly + answer 是兜底
        #    (即 fallback 文案),则尝试 fallback
        if action.action_type == "answer_directly" and "我可以处理" in (action.answer or ""):
            try:
                return self._fallback.route(user_input, project_state)
            except Exception as exc:
                log.warning("LLM fallback 失败: %s", exc)
                return action  # 用 rule 兜底,不阻断 REPL
        return action
```

### 关键设计

- **Rule-first**:优先走 `RuleBasedIntentRouter`,确定性高、零延迟
- **LLM fallback**:只有 rule 把自然语言归类为「兜底 answer」时才触发
- **fallback 异常不阻断**:LLM flaky / 超时 / 配额超限 → log warning,返回 rule 结果

## 4.8 `_select_router` 工厂

> 对应代码:`src/writer/engine/deps.py`

```python
def _select_router(
    settings: Settings,
    *,
    primary: IntentRouter | None = None,
    agent_registry: AgentRegistry | None = None,
) -> IntentRouter:
    rule = primary or RuleBasedIntentRouter()
    if settings.has_api_key:
        return CompositeRouter(
            primary=rule,
            fallback=LlmIntentRouter(settings, agent_registry=agent_registry),
        )
    return rule
```

**配置了 API key** → `CompositeRouter(rule, llm)`;**未配置** → 只用 `RuleBasedIntentRouter`(零 LLM 调用)。

## 4.9 完整数据流:自然语言 vs 斜杠命令

### `/大纲 穿越到唐朝`

```
user_input = "/大纲 穿越到唐朝"
   ↓
RuleBasedIntentRouter.route():
  text.startswith("/") → True
  text.split(maxsplit=1)[0] = "/大纲"
   ↓
AgentAction(action_type="run_command", command="/大纲")
   ↓
engine 分派 → directive_registry.get("/大纲") 命中 → _run_directive
```

### `帮我写一个穿越题材的大纲`

```
user_input = "帮我写一个穿越题材的大纲"
   ↓
RuleBasedIntentRouter.route():
  不以 / 开头
  不包含 "伏笔" / "F0"
  ↓
AgentAction(action_type="answer_directly", answer="我可以处理 ...") ← 兜底
   ↓
CompositeRouter.route():
  检测到 answer 是兜底文案 → 调 fallback
   ↓
LlmIntentRouter.route():
  模型读 system prompt + user input
  产出 AgentAction(action_type="run_command", command="/大纲", answer="穿越到唐朝的程序员...")
   ↓
engine 分派 → directive_registry.get("/大纲") 命中 → _run_directive
```

## 4.10 协议一致性约束

修改 `IntentRouter` Protocol 或 `AgentAction` 字段时,**全量 grep 以下位置**:

- 所有 `RuleBasedIntentRouter` 实例化点
- 所有 `LlmIntentRouter.__init__` 调用
- 所有 `RunnerDeps` 的 `PlainDeps` 测试 stub
- 所有 fake `AgentAction(...)` 测试构造

`AgentAction` 是 frozen Pydantic 模型,**不能 `dataclasses.replace(a, command="/init")`**,必须 `a.model_copy(update={"command": "/init"})`。
