## 一、背景与目标

### 1.1 现状

当前 `DESIGN.md §四` 曾按完整命令清单组织能力,但通过与真实落地节奏对照,发现:

- MVP 范围过宽,容易把正文写作、审核、资料管理一次性塞进同一阶段
- `/init` 应承担更完整的项目启动职责:故事梗概、题材选择、目录结构、`AGENT.md` 和初版大纲
- `/目录`、`/骨架` 应作为正文写作前的独立规划阶段,不应和正文生成混在一起
- `/创意`、`/伏笔`、`/大纲` 需要先形成资料闭环,再进入正文写作
- 缺少**命令 × 项目状态**的可用性矩阵和分阶段验收标准

### 1.2 目标

在不破坏现有 DESIGN.md 总架构的前提下:

1. 将 MVP 从"一次性补齐全部命令"调整为**三阶段交付路线**
2. 阶段一先打通基础 REPL、项目初始化、大纲落盘和目录生成
3. 阶段二补齐创意库、伏笔库、大纲再生成和章节骨架
4. 阶段三再进入正文写作、自动审核修复和分卷级人工确认
5. 建立项目状态机,明确各命令的可用性,避免非法调用

### 1.3 非目标

本期**不**涉及:

- 子代理职责与 Skill 实现(归 DESIGN.md §五、§六)
- 审核流程细节(归 DESIGN.md §八)
- 项目目录结构(归 DESIGN.md §一,不变)
- 多项目、Web UI、外部知识库(归 DESIGN.md §十)

## 三、Section 1 — 分阶段 MVP 命令路线

### 3.1 阶段一 MVP:项目初始化与规划闭环

| 命令 | 目标 | 结果 |
| ---- | ---- | ---- |
| `/帮助` | 展示基础命令与当前阶段能力 | 用户知道可用命令 |
| `/退出` | 退出 REPL | 会话正常结束 |
| `/init` | 输入故事梗概,选择题材 | 生成 `AGENT.md`、项目目录,并创建故事大纲 |
| `/目录` | 根据大纲生成故事全文分卷目录 | 生成卷-章结构,为后续骨架和写作提供章节定位 |

阶段一的验收标准:用户可以从空目录进入 REPL,完成 `/init` 和 `/目录`,得到一个有题材定位、`AGENT.md`、项目目录、大纲和分卷目录的小说项目。

### 3.2 阶段二 MVP:创意、伏笔、大纲与骨架

| 命令 | 目标 | 结果 |
| ---- | ---- | ---- |
| `/创意` | 在创意库中新增一条创意线索 | 创意进入项目资料,可被大纲生成读取 |
| `/伏笔` | 在伏笔库中新增一条伏笔 | 伏笔进入项目资料,可被大纲和写作读取 |
| `/大纲` | 根据创意库、伏笔库生成或重生成故事大纲 | 更新正典大纲 |
| `/骨架` | 根据大纲和目录生成每一章开头、结尾 | 得到全书章节骨架,为正文写作提供边界 |

阶段二的验收标准:用户可以不断补充创意和伏笔,再生成更完整的大纲,并基于大纲与目录得到每章开头/结尾骨架。

### 3.3 阶段三 MVP:分卷写作、审核与自动修复

| 命令 | 目标 | 结果 |
| ---- | ---- | ---- |
| `/创作` | 编写某一卷或某一章节内容;默认从当前卷开始 | 写完一卷后停下,进入审核与自动修复;完全通过审核后结束,下一卷需用户手动开启 |
| `/审核` | 分别对大纲、骨架、正文进行审核 | 产出审核报告,并驱动正文自动修复闭环 |

阶段三的验收标准:系统可以按卷推进正文写作,每卷完成后自动审核和修复,但不会自动进入下一卷,必须由用户显式发起下一轮 `/创作`。

> 当前实现以 `/创作` 作为写章节工作流入口；早期文档中的 `/写作` 属于旧命名。若后续要兼容旧命令,应在 router、状态机、帮助系统中显式增加别名。

### 3.4 预留命名空间(本期不实现)

| 命名空间 | 后续用途 | 示例命令 |
| -------- | -------- | -------- |
| `/设.*` | 会话级参数 | `/设.模型 deepseek-v3`、`/设.字数 5000`、`/设.风格 简洁` |
| `/工具.*` | 高级工具 | `/工具.导入 <文件>`、`/工具.导出 epub`、`/工具.撤回`、`/工具.重试` |
| `/代理.*` | 子代理直连 | `/代理.编剧 关于 F003...`、`/代理.校对`、`/代理.历史` |

**为什么预留**:即使本期不实现,提前占位可避免后续命令名冲突,降低用户记忆负担。

### 3.5 不在三阶段 MVP 内的命令

`/reset`、`/查看`、`/搜索`、`/字数统计`、`/人物卡片`、`/世界观构建`、`/史实`、`/修订`、`/子代理` 等命令不纳入前三阶段 MVP 验收范围,后续按需要进入 v2 或专题规格。

## 四、Section 2 — 命名、子命令、参数语法

### 4.1 命名约定

- **中文为主名**,英文为**可选别名**
- 一个命令最多 2 个别名,**别名仅给高频命令**(`/创作`、`/大纲`、`/审核` 等)
- 例:`/骨架` 不给英文(中文更直观);`/创作` 可留 `/write` 别名(高频、跨平台友好)
- 所有别名必须在帮助系统(§6.5)中可被解析

### 4.2 子命令模式(动词驱动)

**通用模式**(适用于资料新增类命令):

| 调用 | 行为 |
| ---- | ---- |
| `/创意 <文本>` | 向创意库追加一条创意线索 |
| `/伏笔 <文本>` | 向伏笔库追加一条伏笔 |
| `/大纲` | 根据当前创意库、伏笔库和已有项目资料生成或重生成大纲 |
| `/骨架` | 根据大纲和目录生成每章开头、结尾 |

阶段二前不要求提供复杂子命令;先保证资料可追加、正典可再生成、骨架可一次性落盘。

**例外**(默认行为 ≠ 显示):

| 命令 | 默认行为 | 子动词 |
| ---- | -------- | ------ |
| `/init` | 初始化项目、选择题材、生成 `AGENT.md`、项目目录和初版大纲 | 无 |
| `/目录` | 根据大纲生成全文分卷目录 | `rewrite`/`view` |
| `/大纲` | 根据创意库、伏笔库生成或重生成故事大纲 | `rewrite`/`view` |
| `/骨架` | 生成每一章开头、结尾 | `rewrite`/`view` |
| `/创作` | 写作当前卷或指定卷/章节 | 卷/章定位 |
| `/审核` | 审核大纲、骨架或正文 | `大纲`/`骨架`/`正文` |

### 4.3 参数解析

| 命令 | 参数语法 | 示例 |
| ---- | -------- | ---- |
| `/init` | 故事梗概 + 题材选择 | `/init """..."""` 后选择题材 |
| `/目录` | 可选 `rewrite`/`view` | `/目录` / `/目录 rewrite` |
| `/创意` | 创意线索文本 | `/创意 主角其实误解了师父遗言` |
| `/伏笔` | 伏笔文本 | `/伏笔 F003 玉簪真实来历` |
| `/大纲` | 可选 `rewrite`/`view` | `/大纲` / `/大纲 rewrite` |
| `/骨架` | 可选 `rewrite`/`view` | `/骨架` / `/骨架 view` |
| `/创作` | 可选卷/章定位 | `/创作` / `/创作 卷一` / `/创作 1.3` |
| `/审核` | 审核对象 | `/审核 大纲` / `/审核 骨架` / `/审核 正文` |

**卷/章定位优先级**:`卷一` → `1.3` 数字 ID → `卷X第Y章` 中文 ID → 标题匹配。`/创作` 无参数 = 从当前卷开始。

**冲突报错策略**:若三种方式解析出**不同章节**(如 `1.3` 指向「夜奔」但 `夜奔` 也在 1.7 出现),REPL 报错并列出候选,要求用户用更精确的形式(优先数字 ID)。

### 4.4 多行输入:`"""..."""` 包裹

```
> /init """
核心创意第一段...
核心创意第二段...
核心创意第三段。
"""
```

- **首尾 `"""` 必须在行首**,防止误判
- 显式起止,不和单行命令混淆
- 适合脚本化(`/init << 'EOF' ... EOF`)

### 4.5 二次确认(破坏性操作)

需 `y/N` 确认:

| 命令 | 原因 |
| ---- | ---- |
| `/大纲 rewrite` | 覆盖已有大纲 |
| `/目录 rewrite` | 覆盖已有目录 |
| `/骨架 rewrite` | 覆盖已有章节骨架 |
| `/创作` 跨卷继续 | 防止系统自动推进到下一卷 |

非破坏操作无确认:`/帮助`、`/退出`、`/创意`、`/伏笔`、`/审核 大纲`、`/审核 骨架`。

### 4.6 与现有 DESIGN.md §4.3 的对应

DESIGN.md §4.3 已规定:

> 所有命令都**先校验项目状态**(`AGENT.md` 存在性)
> 命令执行的中间过程**实时打印**(用 Rich Live 渲染流式 LLM 输出)
> 命令完成后**写入 AGENT.md 对应字段** + 在 `大纲/大纲.md` 顶部追加修订记录

**本次精修**:
- 「先校验项目状态」 → 具体化为 §5 状态机
- 「实时打印」 → 具体化为 §6.3 输出渲染

## 五、Section 3 — 项目状态机 × 命令可用性

### 5.1 项目状态定义

| 状态 | 触发条件 | 标志文件 |
| ---- | -------- | -------- |
| **S0 待启动** | 无 `AGENT.md` | — |
| **S1 初始化** | 完成 `/init` | `AGENT.md` + `大纲/大纲.md` |
| **S2 大纲拟定中** | 完成 `/大纲` | `大纲/大纲.md` |
| **S3 框架搭建中** | 完成 `/目录` | `目录/目录.md` |
| **S4 正文编辑中** | `/创作` 产出当前卷/章草稿 | `正文草稿/*.md` 或 `正文/*.md` |
| **S5 审核中** | 当前卷正文审核与自动修复通过 | 审核报告 + 正文定稿 |

状态转换:

```
S0 ─/init→ S1 ─/大纲→ S2 ─/目录→ S3 ─/创作→ S4 ─/审核+自动修复→ S5
                              ▲                         │
                              └──── 用户手动开启下一卷 ──┘
```

### 5.2 命令 × 状态矩阵

(✓ 可用 / △ 有条件 / — 不可用)

| 命令 | S0 | S1 | S2 | S3 | S4 | S5 |
| ---- | -- | -- | -- | -- | -- | -- |
| `/帮助` `/退出` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/init` | ✓ | — | — | — | — | — |
| `/目录` | — | ✓ | △¹ | △¹ | △¹ | △¹ |
| `/创意` `/伏笔` | — | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/大纲` | — | ✓ | ✓ | △² | △² | △² |
| `/骨架` | — | — | ✓ | △¹ | △¹ | △¹ |
| `/创作` | — | — | — | ✓ | △³ | △³ |
| `/审核 大纲` | — | ✓ | ✓ | ✓ | ✓ | ✓ |
| `/审核 骨架` | — | — | — | ✓ | ✓ | ✓ |
| `/审核 正文` | — | — | — | — | ✓ | ✓ |

**条件说明**:

1. △¹ —— 已存在目录/骨架时,需 `rewrite` 或 `view`
2. △² —— 已有骨架或正文后重生成大纲会影响后续资料,需确认
3. △³ —— 默认只写当前卷;进入下一卷必须由用户再次显式调用 `/创作`

### 5.3 错误提示模板

```
> /创作
✗ 项目尚未生成章节骨架。请先运行 /骨架(基于大纲和目录生成每章开头结尾)。

> /大纲
✗ 大纲已存在且已有骨架或正文。可选操作:
  - /大纲 rewrite   覆盖重写
  - /大纲 view      查看当前大纲

> /创作
✓ 当前卷已完成并通过审核。请手动确认是否开启下一卷:
  - /创作 卷二
  - /审核 正文
```

### 5.4 状态持久化

- 状态变化时,只更新 `AGENT.md` 的「当前状态」「当前卷」「当前章节」「当前进度」字段
- 不在 `大纲/`、`目录/`、`骨架/` 等文件中隐式记录状态
- Agent 启动读 AGENT.md 一份文件即知状态

## 六、Section 4 — REPL 交互体验

### 6.1 命令历史与补全

| 特性 | 实现 | 优先级 |
| ---- | ---- | ------ |
| ↑/↓ 历史浏览 | 持久化到 `~/.config/writer/history`(最近 1000 条) | MVP |
| Ctrl+R 反向搜索 | 模糊匹配历史 | MVP |
| Tab 补全命令名 | 主名 + 别名 | MVP |
| Tab 补全子动词 | `add`/`edit`/`list` 等 | MVP |
| Tab 补全对象 | 人物名(从 `人物/` 抽取)、章节 ID(从 `目录/` 抽取)、伏笔 ID | MVP |
| 模糊补全 | fzf 风格 | 后期 |

### 6.2 多行输入

| 触发 | 行为 |
| ---- | ---- |
| `"""` 行首起 + `"""` 行首止 | 显式多行(§4.4) |
| 粘贴检测 | 一次性粘贴 > 5 行自动进入多行模式(末行空 Enter 提交) |
| Shift+Enter | 强制换行 |
| Esc | 取消多行输入 |

### 6.3 输出渲染

| 场景 | 渲染 |
| ---- | ---- |
| LLM 流式输出 | Rich Live,打字机效果 |
| 长报告(`/审核`、`/修订`) | Markdown 渲染 + 自动分页(`q` 退出,`/` 搜索) |
| Diff(校对、修订) | 红色删除线 + 绿色新增,并排显示 |
| 章节正文 | 标题层级、引用块、强调 |
| 字数统计 | Rich Table,分卷分章明细 |

### 6.4 中断与进度

- **Ctrl+C** —— 中断当前 LLM 调用,半成品可丢弃或保存草稿(配置项,默认丢弃)
- **长操作 spinner** —— `/创作`、`/审核`、`/大纲`、`/骨架` > 3 秒显示 spinner
- **流式 vs 一次性** —— 默认流式,`/设.流式 false` 可关闭(本期 `/设.*` 未实现,作为环境变量配置)

### 6.5 帮助系统

**`/帮助` 总览**:

```
可用命令(分组):

【阶段一】/帮助 /退出 /init /目录
【阶段二】/创意 /伏笔 /大纲 /骨架
【阶段三】/创作 /审核

输入 /帮助 <命令> 查看详情(如 /帮助 创作)
```

**`/帮助 创作` 详情**:

```
/创作 [卷/章定位]
  用途:写作某一卷或章节正文
  参数:
    卷/章定位 可选,支持:
      - 卷一
      - 1.3(卷1第3章)
      - 卷一第三章
      - 第一章·夜奔(标题匹配)
      - 无参数 = 当前卷
  可用状态:S3(有骨架)及之后
  停止条件:写完当前卷后停下,自动审核修复;下一卷需用户手动开启
  相关命令:/审核 正文
  示例:
    /创作
    /创作 卷一
    /创作 1.3
```

3. `/目录`:根据大纲生成故事全文分卷目录

**阶段二**

1. `/创意`:向创意库新增创意线索
2. `/伏笔`:向伏笔库新增伏笔
3. `/大纲`:基于创意库、伏笔库生成或重生成故事大纲
4. `/骨架`:基于大纲和目录生成每一章开头、结尾

**阶段三**

1. `/创作`:写作某一卷或章节,默认从当前卷开始;写完一卷后停下审核
2. `/审核`:分别审核大纲、骨架、正文
3. 审核后自动修复,完全通过后结束;下一卷必须由用户手动开启
