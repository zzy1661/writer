## 四、Section 2 — 命名、子命令、参数语法

### 4.1 命名约定

- **中文为主名**,英文为**可选别名**
- 一个命令最多 2 个别名,**别名仅给高频命令**(`/写`、`/大纲`、`/审核` 等)
- 例:`/人物` 不给英文(用得少,中文更直观);`/写` 留 `/write` 别名(高频、跨平台友好)
- 所有别名必须在帮助系统(§6.5)中可被解析

### 4.2 子命令模式(动词驱动)

**通用模式**(适用于档案类命令):

| 调用 | 行为 |
| ---- | ---- |
| `/人物` | 列出所有人物(默认 list) |
| `/人物 林远图` | 显示「林远图」档案(默认 show) |
| `/人物 add 林远图 --age 60 --tag 师父` | 新增 |
| `/人物 edit 林远图 --age 62` | 修改 |
| `/人物 remove 林远图` | 删除(需二次确认) |

**默认行为 = 显示**的适用命令:`/人物`、`/世界观`、`/伏笔`、`/查看`(默认列出所有可查看对象)。

**例外**(默认行为 ≠ 显示):

| 命令 | 默认行为 | 子动词 |
| ---- | -------- | ------ |
| `/大纲` | 生成新大纲(S1) / 报错(S2+) | `rewrite`/`append`/`view` |
| `/目录` | 生成新目录(S2) / 报错(S3+) | `rewrite`/`append`/`view` |
| `/改` | 重写整章 | `all`/`tail`/`head`/`section <N>` |
| `/修订` | 列出待并入意见 | `view <id>`/`apply <id>`/`drop <id>` |
| `/init` `/reset` | 执行命令 | 无 |
| `/写` | 写下一章 | 章节定位 |

### 4.3 参数解析

| 命令 | 参数语法 | 示例 |
| ---- | -------- | ---- |
| `/写` | 可选章节定位 | `/写 1.3` / `/写 卷一第三章` / `/写 第一章·夜奔` / `/写`(写下一章) |
| `/查看` | 对象定位 | `/查看 大纲` / `/查看 人物 林远图` / `/查看 1.3` / `/查看 F003` |
| `/搜索` | 关键词(空格分隔多关键词,AND) | `/搜索 信物 林远图` |
| `/init` | `"""..."""` 多行 | 见 §4.4 |
| `/创意` `/伏笔` `/修订` | 可选 `"""..."""` 多行 | 见 §4.4 |
| `/改` | 章节定位 + 范围子动词 | `/改 1.3 tail`、`/改 1.3 section 3` |

**章节定位优先级**:`1.3` 数字 ID → `卷X第Y章` 中文 ID → 标题匹配。无参数 = 顺序下一章。

**冲突报错策略**:若三种方式解析出**不同章节**(如 `1.3` 指向「夜奔」但 `夜奔` 也在 1.7 出现),REPL 报错并列出候选,要求用户用更精确的形式(优先数字 ID)。

### 4.4 多行输入:`"""..."""` 包裹

```
> /init """
核心创意第一段...
核心创意第二段...
核心创意第三段。
"""
```

- **首尾 `"""` 必须在行首**,防止误判
- 显式起止,不和单行命令混淆
- 适合脚本化(`writer init << 'EOF' ... EOF`)

### 4.5 二次确认(破坏性操作)

需 `y/N` 确认:

| 命令 | 原因 |
| ---- | ---- |
| `/reset` | 整项目重置(会备份为 `AGENT.bak.<timestamp>.md`) |
| `/大纲 rewrite` | 覆盖已有大纲 |
| `/人物 remove` `/世界观 remove` `/伏笔 remove` | 删除档案 |
| `/修订 apply` | 并入正典(影响后续所有写作) |
| `/批量` 跨度 > 5 章 | 长操作 |

非破坏操作无确认:`/写`、`/续写`、`/审核`、`/查看`、`/搜索`、`/人物 add`(append)。

### 4.6 与现有 DESIGN.md §4.3 的对应

DESIGN.md §4.3 已规定:

> 所有命令都**先校验项目状态**(`AGENT.md` 存在性)
> 命令执行的中间过程**实时打印**(用 Rich Live 渲染流式 LLM 输出)
> 命令完成后**写入 AGENT.md 对应字段** + 在 `大纲/大纲.md` 顶部追加修订记录

**本次精修**:
- 「先校验项目状态」 → 具体化为 §5 状态机
- 「实时打印」 → 具体化为 §6.3 输出渲染

## 五、Section 3 — 项目状态机 × 命令可用性

### 5.1 项目状态定义

| 状态 | 触发条件 | 标志文件 |
| ---- | -------- | -------- |
| **S0 未初始化** | 无 AGENT.md | — |
| **S1 已初始化** | AGENT.md 已创建 | `AGENT.md` |
| **S2 有大纲** | 完成 /大纲 | `大纲/大纲.md` |
| **S3 有目录** | 完成 /目录 | `目录/目录.md` |
| **S4 写作中** | 至少一章在草稿或正文 | `正文草稿/*.md` 或 `正文/*.md` |
| **S5 完成** | 所有章节定稿(隐式,无命令触发) | 仅 `正文/*.md` |

状态转换:

```
S0 ─/init→ S1 ─/大纲→ S2 ─/目录→ S3 ─/写→ S4 ─...→ S5
└───────────────/reset→ S0(任意状态回退,带备份)
```

| `/大纲` | — | ✓ | △¹ | △¹ | △¹ | △¹ |

### 5.3 错误提示模板

```
> /写
✗ 项目尚未生成目录。请先运行 /目录(基于大纲生成卷-章列表)。

> /大纲
✗ 大纲已存在。可选操作:
  - /大纲 rewrite   覆盖重写
  - /大纲 append    在末尾追加
  - /大纲 view      查看当前大纲

> /续写
✗ 没有未完成的章节。当前状态:S3(有目录,尚未开始写作)。
  - /写 1.1 开始第一章
  - /查看 大纲 浏览规划
```

### 5.4 状态持久化

- 状态变化时,只更新 `AGENT.md` 的「当前状态」「当前章节」「当前进度」字段
- 不在 `大纲/`、`目录/`、`人物/` 等文件中隐式记录状态
- Agent 启动读 AGENT.md 一份文件即知状态

## 六、Section 4 — REPL 交互体验

### 6.1 命令历史与补全

| 特性 | 实现 | 优先级 |
| ---- | ---- | ------ |
| ↑/↓ 历史浏览 | 持久化到 `~/.config/writer/history`(最近 1000 条) | MVP |
| Ctrl+R 反向搜索 | 模糊匹配历史 | MVP |
| Tab 补全命令名 | 主名 + 别名 | MVP |
| Tab 补全子动词 | `add`/`edit`/`list` 等 | MVP |
| Tab 补全对象 | 人物名(从 `人物/` 抽取)、章节 ID(从 `目录/` 抽取)、伏笔 ID | MVP |
| 模糊补全 | fzf 风格 | 后期 |

### 6.2 多行输入

| 触发 | 行为 |
| ---- | ---- |
| `"""` 行首起 + `"""` 行首止 | 显式多行(§4.4) |
| 粘贴检测 | 一次性粘贴 > 5 行自动进入多行模式(末行空 Enter 提交) |
| Shift+Enter | 强制换行 |
| Esc | 取消多行输入 |

### 6.3 输出渲染

| 场景 | 渲染 |
| ---- | ---- |
| LLM 流式输出 | Rich Live,打字机效果 |
| 长报告(`/审核`、`/修订`) | Markdown 渲染 + 自动分页(`q` 退出,`/` 搜索) |
| Diff(校对、修订) | 红色删除线 + 绿色新增,并排显示 |
| 章节正文 | 标题层级、引用块、强调 |
| 字数统计 | Rich Table,分卷分章明细 |

### 6.4 中断与进度

- **Ctrl+C** —— 中断当前 LLM 调用,半成品可丢弃或保存草稿(配置项,默认丢弃)
- **长操作 spinner** —— `/写`、`/审核`、`/大纲` > 3 秒显示 spinner
- **流式 vs 一次性** —— 默认流式,`/设.流式 false` 可关闭(本期 `/设.*` 未实现,作为环境变量配置)
