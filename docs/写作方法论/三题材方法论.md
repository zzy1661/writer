不同题材的长篇，差别往往不在「会不会用三幕式」，而在 **读者契约、冲突类型、信息密度、节奏规则和修订标准** 不同。下面按言情、玄幻、历史三类，分别说常见写作方法与理论，并对比差异。

---

## 一、言情（Romance）

### 读者契约（隐含理论）

言情有一条很硬的底层约定（RWA 传统 + 网文变体）：

- **核心问题**：这两个人（或几对人）能否在一起 / 如何在一起
- **情感弧线** 往往比「事件弧线」更重要
- **HEA/HFN**：主流言情要求 Happy Ending 或至少 Happy For Now（网文里常见「1v1」「双洁」等变体规则）

### 常用结构理论

| 理论                       | 在言情里的用法                                               |
| -------------------------- | ------------------------------------------------------------ |
| **情感节拍（Beat Sheet）** | 相遇 → 吸引 → 误会/阻碍 → 分离危机 → 表白/和解 → 结局        |
| **人物谎言/需求（内弧）**  | 主角的情感创伤、防御机制，比外部反派更驱动剧情               |
| **双主角/多 POV**          | 限知视角切换制造「信息差甜/虐」                              |
| **障碍分级**               | 外部障碍（家族、身份）+ 内部障碍（不信任、自卑）必须同时存在 |

### 规划方法

- **CP 设定表**：人设互补点、雷点、关系阶段目标
- **感情线时间轴**：比「世界大事」更细，常按「好感度阶段」拆章
- **糖/虐比配置**：甜宠、虐恋、先婚后爱等子类型决定节奏模板

### 节奏与钩子

- **高密度情感场景**：对话、肢体距离、误会是「章内爽点」
- **章末钩子**：误会加深、第三者出现、秘密将揭穿
- 篇幅常集中在 **关系转折**，而非世界观展开

### 修订重点

- 情感逻辑是否自洽（「他为何突然不爱了」）
- 人设是否 OOC（Out of Character）
- 配角是否抢戏、男主/女主戏份是否失衡

### 和你们项目的关联

设计里的 **人物档案、梗概、避雷设定** 对言情尤其关键；`StoryConsultant` 若服务言情，prompt 应强化 **关系阶段 + 情感障碍**，而不是四幕战争结构。

## 二、玄幻（Fantasy / 中文网文「玄幻」）

### 读者契约

- **升级与变强** 可见、可量化（境界、功法、法宝）
- **地图/副本/势力** 递进，读者要持续获得「新世界感」
- **爽点** 明确：打脸、夺宝、突破、收小弟、装逼反转

### 常用结构理论

| 理论                       | 在玄幻里的用法                                             |
| -------------------------- | ---------------------------------------------------------- |
| **英雄之旅**               | 废柴开局 → 奇遇 → 试炼 → 宗门/王国级对抗 → 位面/天道级终局 |
| **力量体系理论**           | 修炼等级必须清晰、可预期、有代价（网文「战力体系」）       |
| **地图升级（Escalation）** | 村 → 城 → 国 → 大陆 → 上界，每一阶换地图、换对手层级       |
| **七点式/卷纲**            | 大 Boss 分卷打；每卷有小 Boss + 卷末大高潮                 |

### 规划方法

- **世界观圣经**：力量规则、资源、势力、地理（设定即剧情）
- **升级表**：主角何时突破、拿什么功法、欠什么因果
- **副本/事件清单**：秘境、拍卖会、宗门大比等「标准桥段」
- **伏笔-回收表**：传承、前世、血脉、老爷爷（你们 `foreshadow_query` 的方向）

### 节奏与钩子

- **小高潮密、大高潮疏**：几章一小爽，一卷一大爽
- **黄金开篇**：尽快展示金手指或核心设定差异
- **章末**：突破在即、强敌登场、宝物现世

### 修订重点

- **战力崩坏**、等级名乱、前后设定矛盾
- **灌水**：升级无代价、地图换皮无新意
- **人物工具化**：配角只为送经验

### 和你们项目的关联

四幕 MVP 对玄幻 **偏粗**；更自然的是 **分卷 + 境界节点 + 地图表**。`/目录` 宜按「卷」组织，大纲宜写「本卷核心矛盾 + 升级目标」，而非纯文学幕结构。

## 三、历史（Historical / 架空历史）

### 读者契约

- **时代感**：称谓、礼制、科技水平、社会结构可信
- **历史题材** 读者常期待 **史实尊重** 或 **偏离有标注**
- **权谋/格局** 常重于个人魔法式开挂

### 常用结构理论

| 理论                      | 在历史里的用法                               |
| ------------------------- | -------------------------------------------- |
| **史实锚点（Anchoring）** | 重大事件、人物卒年、官制、战争时间线作硬约束 |
| **虚构边界理论**          | 大事件不可乱改，小人物、私情、野史可虚构     |
| **群像/权谋结构**         | 多势力博弈，非单一主角升级线                 |
| **时代悲剧结构**          | 个人命运被结构碾压（现实主义历史向）         |

### 规划方法

- **年表 + 人物表 + 官制表**（你们设计里的 `史实/` 目录）
- **大事年表对齐大纲**：哪年发生什么、主角站在哪一派
- **考证备忘**：哪些细节已核实、哪些 deliberate fiction
- **语言分层**：对白古今度、叙述语体统一策略

### 节奏与钩子

- 钩子常来自 **朝局变动、战争、诏狱、科举、联姻**
- 节奏可慢于玄幻，但 **信息密度**（势力关系）要高
- 不宜频繁「超时代」知识降维打击（除非穿越设定且需自洽）

### 修订重点

- **史实错误**（官职、称谓、年代、地理）
- **现代口语穿帮**
- **架空却缺乏内部一致性**

### 和你们项目的关联

设计里 **HistoryConsultant**、历史题材才加载 `史实/`、审核必含史实检查——这正是历史题材 **方法论在工程上的落地**：史实是独立 QA 维度，不是普通校对能替代的。

## 四、三题材横向对比（一张表）

| 维度               | 言情                 | 玄幻                 | 历史                 |
| ------------------ | -------------------- | -------------------- | -------------------- |
| **核心驱动力**     | 情感关系             | 力量/目标升级        | 时代结构与权谋       |
| **主结构**         | 情感节拍             | 地图+等级递进        | 年表+势力线          |
| **设定权重**       | 中（人设>世界观）    | 极高（体系即剧情）   | 高（时代规则）       |
| **爽点类型**       | 甜、虐、反转表白     | 突破、打脸、夺宝     | 翻盘、献策、战争胜负 |
| **典型障碍**       | 误会、身份、内心创伤 | 强敌、天劫、资源匮乏 | 制度、派系、历史洪流 |
| **POV**            | 双主角/多视角常见    | 单主角为主           | 群像、多线常见       |
| **伏笔类型**       | 身份秘密、前任       | 传承、血脉、幕后黑手 | 史书未载密谋、假史   |
| **修订硬指标**     | 情感逻辑、OOC        | 战力、设定一致性     | 史实、时代语感       |
| **Agent 专属角色** | 人设/关系顾问        | 世界观/战力顾问      | **历史顾问**         |

## 五、同一「底层方法论」在三题材里的不同用法

### 三幕/四幕

- **言情**：三幕常映射 **相识—阻碍—在一起**
- **玄幻**：三幕常映射 **一卷或一个地图阶段**，全书重复多轮
- **历史**：三幕可能对应 **一次战争/一次政变**，全书多条三幕嵌套

### 雪花法

- 言情：雪花中心是 **一句话 CP 矛盾**（「仇人之子爱上她」）
- 玄幻：中心是 **金手指 + 终极目标**（「废柴要弑天道」）
- 历史：中心是 **时代命题**（「小吏如何在党争中保全家族」）

### Plan-Write-Review

- 言情：Review 看 **情感节奏、人设**
- 玄幻：Review 看 **升级合理性、爽点密度**
- 历史：Review 看 **史实 + 权谋逻辑 + 语言**

## 六、对 writer-agent 的启示（若按题材分支）

若未来做 **GenreLocator**（设计文档已提），每种题材可挂不同「默认方法论包」：

| 题材 | 建议默认结构  | 大纲字段                | 目录逻辑         | 必载子代理/Skill       |
| ---- | ------------- | ----------------------- | ---------------- | ---------------------- |
| 言情 | 情感节拍      | CP 表、关系阶段         | 按感情线阶段分章 | 人设一致性、避雷       |
| 玄幻 | 分卷+升级节点 | 境界表、地图、本卷 Boss | 按卷+副本        | 战力/设定校验、伏笔    |
| 历史 | 年表+势力     | 史实锚点、虚构标注      | 按年/按事件      | **历史顾问**、官制词表 |

当前项目的 **固定四幕** 更像「通用占位」；按题材分化后，言情可能默认 **8～12 个情感节拍**，玄幻默认 **分卷纲**，历史默认 **大事年表 + 幕**，会更贴领域实践。
