# diff 输出格式参考

`/改` 工具在生成 diff 模式输出时的格式约定。优先使用 unified diff 格式，与 Git 一致。

## Unified Diff 格式

```diff
--- manuscript/chapter-1.md
+++ manuscript/chapter-1.md
@@ -10,7 +10,7 @@

 主角抬头，看见……

-他原本是个程序员。
+他原本是个穿越而来的现代程序员。

 此刻他站在长安街头……
```

每段 diff 以 `@@` 行标记上下文行号（与 Git diff 一致）。

## 简洁变更摘要格式

当用户只需知道"改了什么"而不是逐字对比时，使用以下摘要：

```markdown
# 章节 1 修改摘要

## 修改点 1（10-12 行）
- 原句：「他原本是个程序员。」
- 新句：「他原本是个穿越而来的现代程序员。」
- 修改意图：补全主角身份（用户原指令：补全主角身份背景）

## 修改点 2（25-28 行）
- 原句：「……」
- 新句：「……」
- 修改意图：……
```

适合：
- 用户只想 review 改动点
- 章节很长，逐字对比难以阅读

## Side-by-side 格式

行级对比，便于视觉差异检测：

```markdown
| 行 | 原句 | 新句 | 备注 |
| --- | --- | --- | --- |
| 10 | 他原本是个程序员。 | 他原本是个穿越而来的现代程序员。 | 补全身份 |
| 25 | …… | …… | …… |
```

适合：
- 中等长度章节（< 100 行）
- 修改密集、改动较大

## 默认输出策略

| 章节长度 | 默认输出 |
| --- | --- |
| < 30 行 | 全文 diff（unified） |
| 30-100 行 | 简洁变更摘要 |
| > 100 行 | 简洁变更摘要 + 仅显示修改点上下文 |

用户可通过 `/改 ... --format=diff|summary|side-by-side` 显式覆盖。
